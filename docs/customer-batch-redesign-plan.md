# 客戶批號與訂單明細改版設計

更新日期：2026-07-19

## 一、改版目標

將「訂單主表管理的訂單明細」整理成可獨立追蹤的業務單位，並新增側邊欄「客戶批號」全域入口，讓使用者可以從客戶批號直接查看下游工單、庫存、出貨與退貨／沖銷狀態。

本改版不改變客戶批號的業務意義：客戶批號仍是客戶隨貨或表單提供的原始批號，不由系統自動填造。

## 二、正式名詞與資料責任

### 訂單主表

- 資料表：`orders`
- 對外識別：`order_number`
- 一張訂單可包含多筆訂單明細。

### 訂單明細／客戶批號明細

- 資料表：`order_items`
- 目前已存在的客戶欄位：`customer_batch_number`
- `customer_batch_number`：客戶提供的原始批號，可暫時為空並顯示「待補」；禁止以系統編號取代。
- 新增系統識別欄位：`order_item_number`
- 格式：`{order_number}-L{兩位明細序號}`

範例：

```text
ORDER-20260719-0001-L01
ORDER-20260719-0001-L02
```

`order_item_number` 必須能直接辨識所屬訂單主表，建立後保持穩定且不可任意修改；目前預期每張訂單不超過 99 筆明細，超過時另行處理例外策略。

### 生產工單

- 資料表：`work_orders`
- 關聯來源：`work_orders.order_item_id`
- 不在工單另建一套批號主檔。
- 工單畫面同時顯示 `order_number`、`order_item_number` 與 `customer_batch_number`。

### 下游資料

工單、庫存、出貨、退貨／沖銷均沿用 `order_item_id` 或既有來源鏈追溯，不以客戶批號文字作為唯一關聯鍵。

## 三、側邊欄「客戶批號」入口

側邊欄顯示所有訂單明細，不限目前開啟的訂單主表。資料來源直接使用 `order_items`，不建立重複的客戶批號主表。

列表建議欄位：

| 欄位 | 說明 |
|---|---|
| 訂單明細編號 | `ORDER-...-L01`，主要系統識別碼 |
| 客戶批號 | 客戶原始批號，未提供時顯示「待補」 |
| 訂單主表 | 顯示 `order_number` |
| 客戶 | 客戶名稱 |
| 受篩產品 | 訂單明細產品 |
| 工單 | 工單數量與目前狀態 |
| 庫存 | 有效庫存／待入庫摘要 |
| 出貨 | 配貨、已出貨或退貨摘要 |
| 操作 | 查看明細、開啟訂單、開啟流程 |

搜尋至少支援：

- 訂單明細編號
- 客戶批號
- 訂單主表編號
- 客戶名稱
- 工單編號

客戶批號不保證全域唯一，搜尋與顯示必須搭配客戶、訂單或系統明細編號辨識。

## 四、客戶批號工作區

點擊列表項目後進入工作區，頁首固定顯示：

```text
客戶：○○公司
訂單：ORDER-20260719-0001
訂單明細：ORDER-20260719-0001-L01
客戶批號：CUSTOMER-BATCH-ABC
產品：M3 × 10
```

工作區頁籤：

1. 基本資料
2. 生產工單
3. 庫存
4. 出貨
5. 退貨／沖銷
6. 流程歷程
7. 附件與圖面

所有下游頁籤都必須能回到原始訂單、訂單明細與客戶批號。

## 五、訂單主表建立流程

### 第一步：建立訂單主表

只處理訂單層級資料：

- 客戶
- 訂單日期
- 預計交期
- 客戶訂單號碼
- 訂單備註

建立成功後產生 `order_number`，留在訂單主表分頁並自動展開新訂單，直接開啟與客戶批號模組完全相同的完整「新增訂單細項（客戶批號）」視窗。

側邊欄「客戶批號」則維持獨立的全域一覽分頁；建立明細時不切換或覆蓋全域一覽，使用者可在兩個分頁間切換瀏覽。

### 第二步：管理訂單明細

訂單明細工作區採「工作清單」互動形式，但正式名稱使用「訂單明細清單」，避免與一般 Todo 混淆。訂單主表內嵌明細清單負責新增／編輯目前訂單；側邊欄客戶批號頁負責跨訂單瀏覽與追蹤。兩個入口共用 `modules/order_items.html` 的同一個完整 Modal 與 `js/order_items.js` 的同一套欄位、計算、驗證、附件、載具、篩分服務及儲存邏輯，禁止另建縮水表單。

每筆明細顯示：

- 系統明細編號
- 客戶批號
- 產品
- 重量／支數
- 篩分服務
- 圖號／附件
- 工單狀態
- 庫存狀態
- 出貨狀態

## 六、訂單明細清單操作

```text
[新增客戶批號明細]

ORDER-...-L01 | 客戶批號 | 產品 | 工單 | 庫存 | 出貨 | [編輯] [查看流程]
ORDER-...-L02 | 客戶批號 | 產品 | 工單 | 庫存 | 出貨 | [編輯] [建立工單]
ORDER-...-L03 | 待補     | 產品 | 待建立 | - | - | [編輯] [移除]
```

操作規則：

- 新增：建立一筆訂單明細草稿。
- 儲存：每筆明細獨立儲存，避免整張訂單一起失敗。
- 編輯：修改仍允許修改的明細欄位。
- 建立工單：從該明細直接建立工單。
- 查看流程：直接開啟客戶批號工作區。
- 暫存多筆：允許連續新增多筆草稿，再逐筆或批次儲存。

## 七、刪除與流程守門

刪除按鈕必須依流程狀態變化，不能只使用一般 Todo 的移除語意：

| 狀態 | 可用操作 |
|---|---|
| 草稿、未建立工單 | 儲存、編輯、移除 |
| 已儲存、未建立工單 | 編輯、建立工單、移除 |
| 已建立工單 | 查看工單、受限編輯；禁止直接刪除 |
| 已產生庫存 | 查看工單／庫存；禁止直接刪除，走沖銷或作廢 |
| 已配貨或已出貨 | 查看流程；禁止直接刪除，走退貨／沖銷 |

後端仍以 workflow guard 為最終判斷，前端只負責顯示影響與引導合法流程。
