# 拆分工單與不良品管制討論紀錄（2026-05-23）

## 目標

- 支援同一張生產工單可分派到多台機台並行生產。
- 保留原本一般工單流程，不影響現行單機台作業。
- 建立可追溯、可分析機台能力的不良品資料模型。

## UI 互動共識

1. 生產工單編輯 Modal 增加工單類型：一般工單 / 拆分工單。
2. 一般工單：維持目前流程。
3. 拆分工單：切換到拆分專用場景（建議同一 overlay 內做模式切換，避免雙層 modal 焦點與可用性問題）。
4. 拆分場景版型：
   - 左側：機台頁籤清單（新增機台時頁籤同步新增）。
   - 中央：當前機台填寫區（排程、數量、不良項目與數量）。
   - 右側：即時生產統計（分配總量、機台填寫進度、不良合計、差異值）。
5. 待排程工單區：拆分工單需有可視化識別（標籤/顏色）與已排程機台數字提示。
6. 生產工單排程中若主工單拆成多個機台，需使用直覺色彩與「拆分工單」標籤區隔一般工單。

## 不良品管制共識

1. 拆分後每台機台不良資料必須各自填寫，再由主工單合計。
2. 即使某機台零不良，也必須明確填寫 0，不可留空。
3. 主工單只做彙總，不能直接覆寫各機台明細。
4. 管制目的：可追溯到機台層，作為機台生產能力分析依據。

## 即時統計共識

1. 任一機台頁籤資料變更，右側統計需即時更新。
2. 至少顯示：
   - 已配置機台數 / 已完成填寫機台數
   - 各機台生產數量
   - 各機台不良合計與主工單不良總計
3. 統計值需可回推到機台頁籤原始資料，避免黑盒數字。

## MVP 建議範圍（下一階段）

1. 完成工單類型切換與拆分 UI 容器。
2. 完成左側機台頁籤動態增減與右側即時計算。
3. 完成儲存前最低阻擋：
   - 至少 1 台機台
   - 每台不良資料已填（含 0）
   - 各機台數量可加總且通過基本檢核
4. 先以前端資料模型完成互動與驗證，再串接 API。

## 待定決策（下一輪先定案）

1. 拆分分配主單位：已定案為重量優先，支數由內容物淨重與單隻重量反推。
2. 每台機台不良欄位最低必填集合：已定案為確認單上的篩分服務項目，0 不良也需明確填寫。
3. 主工單完工門檻：已定案為全部拆分機台完成才可結案；已完成部分可先入庫/先出貨，但需另表追蹤並在最終完工時結清。

## 實作追蹤

- 詳細計畫：`docs/split-work-order-implementation-plan-2026-05-31.md`
- 生產工單與生產工單排程需一併改造；排程需以機台執行明細為單位，而不是只更新主工單 `machine_id`。
- 部分完工入庫/拆分機台合計不得超過主工單預期內容物淨重，超過時必須硬阻擋。
