# 生產工單製程、庫存與出貨整合實作規格

> 文件狀態：草案，供需求確認與修改
> 建立日期：2026-07-23
> 文件位置：`docs/work-order-production-flow-implementation-spec-2026-07-23.md`
> 適用範圍：生產工單、篩分階段、平行機台、首件檢驗、卡號、載具、不良品包裝、庫存、二次篩分與出貨

## 1. 文件目的

本文件將目前已確認的生產流程收斂成單一實作版本，作為後續 migration、API、Modal、庫存、出貨、報表及測試的共同依據。

修改本文件時，應優先調整「最終決策摘要」，再同步修改後續資料模型、流程、驗證與驗收案例，避免同一規則在不同章節出現衝突。

## 2. 最終決策摘要

| 編號 | 最終決策 |
|---|---|
| D01 | 一張生產工單先走一般生產與篩分流程；有後續需求時才建立並明確標示「二次篩分」，不以新增另一張一般工單表示。 |
| D02 | 趕工時「加開第二機台」代表同一篩分流程增加平行機台，不等於二次篩分。 |
| D03 | 每個篩分階段可有一台或多台機台；每台機台有獨立排程、人員、重量、卡號、載具、首件尺寸、篩分不良、圖片與完成結果。 |
| D04 | 一般篩分良品可直接完成入庫，或以「第二道工序」方式進入二次篩分；一般篩分不良品可直接列為最終不良，或以「放寬標準」方式進入二次篩分。 |
| D05 | 轉入二次篩分的數量不先入庫；只有到達流程終點的良品與不良品才建立正式庫存。 |
| D06 | 機台顯示不良支數與秤重換算不良支數必須同時保存，不得互相覆蓋。 |
| D07 | 機台顯示數量用於製程統計與追溯；有上傳圖片時再供圖片比對。不良品實體庫存以秤重換算支數為準。 |
| D08 | 不良品換算採「不良品實秤重量 × 1000 ÷ 單支重」，四捨五入至整支；後端為唯一計算來源。 |
| D09 | 單支重必須保存完成當下的快照，歷史換算不得因產品主檔日後修改而重新計算。 |
| D10 | 篩分完成圖片不是所有工單的固定必填資料；有上傳圖片時，必須綁定對應的階段、機台與完成結果，供原始數量追溯。 |
| D11 | 良品與不良品都要進入庫存，並且都是可保留、可分配、可出貨、可撤銷追蹤的實體庫存。 |
| D12 | 良品與不良品建立為不同庫存項目；良品使用載具，不良品通常使用塑膠袋。 |
| D13 | 塑膠袋只記錄包／袋數，不計袋重、不列入載具重量；不良品實秤重量直接視為內容物淨重。 |
| D14 | 良品進料載具與出料載具分開記錄；支援原載具出、不同載具出及混合使用。 |
| D15 | 不同載具出料時，原進料載具必須保留後續處置，例如空載具歸還、暫存、損壞或其他。 |
| D16 | 出貨單必須能同時加入良品庫存、不良品庫存及空載具；混合出貨時各自扣除正確庫存與實體包裝。 |
| D17 | 現有 `rescreen_batches` 保留處理已入庫、已出貨或客退後的重篩；不作為同一工單內二次篩分的主要資料模型。 |
| D18 | 既有一般／拆分工單切換退出新流程；新流程一律是一張工單下的一個或多個階段及機台。 |
| D19 | 「第一階段」只作為內部 `stage_sequence = 1` 的排序概念，不出現在一般操作介面；一般流程頁面稱為「生產與篩分」，只有後續流程顯示「二次篩分」。 |
| D20 | 「第二道工序」不是二次篩分前的獨立流程節點，而是二次篩分方式之一；二次篩分方式分為 `second_process` 第二道工序及 `relaxed_standard` 放寬標準。 |
| D21 | 同一工單若同時存在兩種二次篩分方式，必須分成不同二次篩分執行實例，各自保存來源、標準、機台及結果，但共同顯示在「二次篩分」頁面。 |
| D22 | 二次篩分描述的是第二次篩分流程，不代表來源一定是不良品；`second_process`的來源是一般篩分良品，`relaxed_standard`的來源才是一般篩分不良品。 |
| D23 | 第二道工序型二次篩分不得建立不良來源或要求放寬佐證；放寬標準型二次篩分必須使用秤重換算後的不良支數並保留放寬佐證。 |
| D24 | 生產工單有兩個建立入口：由確認單的訂單細項「轉為工單」，或直接在生產工單頁面按「新增工單」。 |
| D25 | 「轉為工單」只是預先帶入來源訂單細項、客戶、品項、數量、服務規格與交期等資料的便利操作；兩個入口必須共用同一套工單模型、建立交易、驗證及後續生產流程，不得形成不同工單類型。 |
| D26 | 是否需要圖片佐證依客戶要求決定；未設定時預設為選填，客戶預設值可由該筆確認單／訂單細項明確覆寫，建立工單時保存要求快照。快照為選填時允許缺圖正常完成，快照為必填時才阻擋完成。 |
| D27 | 卡號是機台累計篩分支數的「裝載參考號」，不是資料主鍵或人工固定識別碼；預設與自行輸入兩種載具模式都依該機台預計支數及載具列數自動計算。 |
| D28 | 生管預設模式使用訂單載具規劃；自行輸入模式由現場調整實際載具配置。載具完成裝載後，現場在對應卡號旁輸入實秤毛重，系統保存載具皮重快照並計算內容物淨重。已有實秤資料的卡號列必須鎖定，後續只重算尚未完成的卡號。 |

## 3. 名詞與資料層級

### 3.1 名詞

- **生產工單**：一筆訂單細項對應的完整篩分工作。
- **工單建立入口**：包含確認單的訂單細項「轉為工單」及生產工單頁面「新增工單」；入口不同不改變工單性質。
- **一般篩分**：工單建立後預設執行的生產與篩分流程；介面不顯示「第一階段」字樣。
- **二次篩分**：描述第二次篩分流程，不是品質判定名稱；來源可以是一般篩分良品，也可以是一般篩分不良品。
- **第二道工序**：二次篩分方式之一；來源為一般篩分良品，表示該批良品依原定下一道篩分要求繼續處理，不屬於不良品重篩。
- **放寬標準**：二次篩分方式之一；來源為一般篩分不良品，表示該批不良品依核准後的放寬標準重新篩分。
- **篩分階段**：資料庫內用於承載一般篩分與二次篩分的內部層級；可用 `stage_sequence` 排序，但不直接作為一般流程的介面名稱。
- **機台執行**：某一篩分階段分配到一台實際機台的作業單位。
- **機台完成結果**：機台結束本次作業後確認的製程數量、秤重數量、檢驗、圖片與輸出。
- **圖片要求快照**：工單建立時由客戶預設與該筆確認單／訂單細項覆寫值解析出的最終圖片要求；至少區分 `optional` 選填與 `required` 必填，歷史工單不受日後客戶設定異動影響。
- **階段輸出**：機台完成後形成的良品流或不良品流。
- **轉流**：將階段輸出直接入庫，或轉入下一個篩分階段。
- **原始不良支數**：機台電腦畫面顯示的不良支數。
- **入庫不良支數**：不良品實秤重量依單支重換算後的支數。
- **進料載具**：客戶批號進入生產時使用的桶、籃或其他載具。
- **出料載具**：良品完成後實際裝載並進入庫存或出貨的載具。
- **不良品包裝**：不良品使用的塑膠袋，以包或袋記錄，不屬於載具。
- **卡號／裝載參考號**：機台累計篩分數量達到此值時，提醒現場完成目前載具裝載並換下一個載具；穩定資料識別使用生產紀錄ID，不使用可重算的卡號值。

### 3.2 資料層級

```text
生產工單
├─ 生產與篩分（預設一般流程；內部 stage_sequence = 1）
│  ├─ 服務與標準快照
│  ├─ 一台或多台機台執行
│  └─ 良品／不良品輸出與後續處置
└─ 二次篩分（有轉流時才建立；內部 stage_sequence = 2）
   ├─ 二次篩分標準快照
   ├─ 一台或多台機台執行
   └─ 最終良品／不良品輸出

每個機台執行包含：
├─ 進料載具分配
├─ 卡號與生產紀錄
├─ 首件尺寸檢驗
├─ 篩分不良明細
├─ 完成結果與圖片
├─ 良品出料載具
└─ 不良品包裝
```

## 4. 正式生產流程

### 4.1 工單建立

工單有兩個建立入口：

1. **確認單轉為工單**：使用者在確認單的訂單細項按「轉為工單」。系統以該筆訂單細項開啟新增工單Modal，並預先帶入可繼承資料，減少重複選擇與輸入。
2. **工單頁面直接新增**：使用者在生產工單頁面按「新增工單」，於Modal內先選擇來源確認單及訂單細項，再由系統帶入相同資料。

兩個入口在送出前及送出後遵守相同規則：

1. 最終都必須綁定一筆有效的來源訂單細項。
2. 預帶或選定來源後，使用相同欄位映射載入客戶、品項、數量、篩分服務、規格、交期及載具等來源資料。
3. 使用相同權限、來源狀態、可轉數量、重複建立及必填欄位驗證。
4. 呼叫同一套工單建立交易，建立相同的生產工單及預設一般生產與篩分流程。
5. 不得以入口判斷工單類型、製程路徑、庫存規則或報表分類。
6. 若需追蹤操作入口，只記錄為稽核事件或建立來源資訊，不得分叉核心業務模型。
7. 使用者選擇第一台機台並設定排程、負責人、校機人員、分配重量及支數。
8. 若需趕工，在「生產與篩分」頁面按「加開機台」，增加同流程的平行機台。
9. 各機台分別配置進料載具、卡號與預計處理量。

### 4.2 一般生產與篩分

每台機台獨立執行：

1. 分配進料載具與內容物。
2. 完成首件尺寸檢驗。
3. 執行篩分並保存卡號、生產重量、時間及人員。
4. 記錄各篩分服務的機台原始不良支數。
5. 依工單圖片要求快照上傳機台電腦畫面；設定為選填時可略過。
6. 秤量不良品重量並由系統換算入庫不良支數。
7. 確認良品數、不良品雙數量、出料載具及不良品袋數。
8. 產生該機台的良品輸出與不良品輸出。

### 4.3 一般篩分輸出處置

良品輸出可選：

- `terminal_good`：一般篩分良品直接完成並建立良品庫存。
- `secondary_screening + second_process`：一般篩分良品以「第二道工序」方式進入二次篩分；這批來源仍是良品，不建立不良品紀錄。

不良品輸出可選：

- `terminal_defect`：一般篩分不良品直接完成並建立不良品庫存。
- `secondary_screening + relaxed_standard`：一般篩分不良品保存放寬標準、原因及客戶同意佐證後，以「放寬標準」方式進入二次篩分。

同一機台的良品與不良品可走不同路徑。轉流數量不得超過該輸出的可用數量。

### 4.4 二次篩分

1. 只有一般篩分存在轉流時才建立或啟用二次篩分。
2. 建立二次篩分時必須指定方式：
   - `second_process`：第二道工序。
   - `relaxed_standard`：放寬標準。
3. 第二道工序方式的來源品質固定為`good`，輸入數量取自一般篩分已確認良品；保存原定下一道篩分要求的服務項目、標準、公差及PPM快照。
4. 放寬標準方式的來源品質固定為`defect`，輸入數量取自一般篩分秤重換算不良支數；保存來源標準、放寬後標準、原因及客戶同意佐證。
5. 二次篩分來源必須可追溯到一般篩分、來源機台、來源結果、來源品質類別及來源數量。
6. 同一工單若同時存在兩種方式，建立兩個獨立二次篩分執行實例，不得共用規格或結果。
7. 每個二次篩分執行實例可重新分配一台或多台機台。
8. 二次篩分機台重複一般流程相同的排程、卡號、載具、檢驗、圖片要求、雙數量及完成流程。
9. 二次篩分完成後的良品及不良品均為流程終點，分別建立正式庫存。

兩種情境的差異：

| 二次篩分方式 | 來源品質 | 來源數量 | 規格 | 放寬佐證 |
|---|---|---|---|---|
| 第二道工序 `second_process` | 一般篩分良品 | 已確認良品支數 | 原定下一道工序規格 | 不需要 |
| 放寬標準 `relaxed_standard` | 一般篩分不良品 | 秤重換算不良支數 | 核准後放寬規格 | 必須 |

兩種方式完成後都會重新產生自己的良品與不良品；第二道工序型雖然來源是良品，仍稱為二次篩分。

### 4.5 工單完成條件

工單必須同時符合下列條件才可完成：

- 所有必要篩分階段均已完成。
- 所有未取消機台均已完成並確認結果。
- 所有階段輸出都已入庫或完成合法轉流。
- 不存在尚未處理的良品／不良品輸出。
- 不存在尚未完成的二次篩分轉入數量。
- 所有終點良品與不良品均已建立庫存來源及庫存交易。
- 圖片要求快照列為必填的必要圖片、首件尺寸及完成確認資料均完整；圖片選填的工單不因缺圖阻擋結案。

## 5. 機台結果與100／99雙數量

### 5.1 必存欄位

每次機台完成結果至少保存：

- 本次投入支數及重量。
- 機台顯示處理總數。
- 機台顯示良品支數。
- 機台顯示不良支數，例如100支。
- 不良品實秤重量。
- 單支重快照。
- 重量換算不良支數，例如99支。
- 不良差異支數，例如 `99 - 100 = -1`。
- 換算規則，例如 `round_half_up`。
- 完成時間與完成者。
- 確認時間與確認者。
- 依圖片要求快照保存零張或多張機台電腦畫面；只有快照列為必填時才要求最低張數。

### 5.2 換算規則

```text
入庫不良支數 = 四捨五入（不良品實秤重量kg × 1000 ÷ 單支重g）
差異支數 = 入庫不良支數 - 機台顯示不良支數
```

規則：

- 前端只顯示即時計算預覽。
- 後端必須以相同公式重新計算。
- 使用者不得直接輸入或覆蓋最終換算支數。
- 塑膠袋重量不計，因此實秤重量直接作為不良品淨重。
- 單支重小於等於0時不得完成。
- 確認後不得因產品主檔單支重異動而重算。

### 5.3 使用目的

| 數值 | 用途 |
|---|---|
| 機台顯示不良支數 | 製程統計、設備畫面核對、品質分析與追溯 |
| 入庫不良支數 | 不良品實體庫存、配貨、出貨及客戶歸還 |
| 差異支數 | 誤差監控、異常提醒及稽核 |
| 機台畫面 | 證明當時設備顯示數值，不能取代結構化數量 |

### 5.4 圖片規則

- 圖片要求由客戶預設值帶入，並允許在確認單／訂單細項針對該批需求明確覆寫。
- 客戶、確認單及訂單細項都沒有設定時，最終要求為 `optional`，不得因缺圖阻擋。
- 工單建立時將解析後的最終要求保存為快照，至少包含圖片類型、是否必填及最低張數。
- 快照為 `optional` 時，沒有上傳圖片屬正常流程，不需要權限、豁免原因或異常紀錄。
- 快照為 `required` 時，對應機台結果達到要求張數前不得確認完成。
- 二次篩分預設沿用工單圖片要求快照；若客戶對二次篩分另有明確要求，建立該二次篩分實例時另存覆寫快照。
- 有上傳的圖片必須綁定機台完成結果，不只綁定工單。
- 圖片需保存檔名、路徑、MIME、大小、上傳者、上傳時間及說明。
- 已確認結果的圖片不可直接刪除；更正應保留原圖並新增更正紀錄。

## 6. 篩分服務與首件檢驗

### 6.1 篩分服務

- 一般篩分由訂單細項帶入服務項目、公差與PPM，建立不可被來源異動影響的規格快照。
- 各機台顯示相同階段的服務規格，但不良數量及備註由各機台獨立填寫。
- `work_order_machine_defects.defect_quantity` 保持「機台畫面原始數量」語意。
- 機台服務不良明細合計應與機台結果的原始不良總數一致；不一致時禁止確認，或由具權限人員填寫差異原因。

### 6.2 二次篩分方式與放寬標準

- 第二道工序與放寬標準都是二次篩分方式，不是二次篩分前後的額外流程節點。
- 第二道工序方式的來源是一般篩分良品，沿用或帶入原定下一道篩分要求，不得標示為不良品重篩。
- 第二道工序方式不需要放寬原因、客戶同意或不良歷史來源。
- 放寬標準方式只能建立新的二次篩分規格，不可覆蓋一般篩分的原始標準。
- 放寬標準方式的來源是一般篩分不良品，轉入支數必須使用秤重換算值。
- 放寬標準方式必須保存來源規格、放寬後規格、放寬原因、客戶同意或佐證及建立者。
- 歷史報表需同時顯示原始與放寬後標準。

### 6.3 首件尺寸

- 首件尺寸屬於機台執行，不屬於整張工單的單一共用欄位。
- 每台機台至少可保存一次首件檢驗。
- 重新校機後可新增下一次檢驗，不可覆蓋前一次結果。
- 保存檢驗輪次、尺寸值、合格判定、檢驗者、檢驗時間及備註。
- 首件未通過時，該機台不得進入正式生產或完成。

## 7. 卡號、進料載具與良品出料載具

### 7.1 卡號

- 卡號屬於機台執行及其生產紀錄。
- 卡號語意是機台累計支數的裝載參考點，不是卡片或資料列的穩定主鍵；資料識別使用 `production_records.id`。
- 不同機台各自計算卡號序列，不得用主工單總支數跨機台共用。
- 生管「預設」與現場「自行輸入」兩種模式都自動計算卡號；兩者差異只在載具配置來源，不在卡號計算方式。
- 計算方式依該機台預計處理支數及目前載具列數平均分配，最後一張卡的參考號不得超過機台預計支數。
- 載具裝載完成後，現場在該卡號列輸入磅秤實際毛重；系統同時保存載具皮重快照，計算並保存內容物實際淨重。
- 尚未輸入實秤重量的卡號可隨載具列增減重新計算。
- 一旦某卡號已有實秤重量、檢驗、入庫或其他實際資料，該列參考號與載具快照鎖定；中途增減載具只重新分配尚未完成的剩餘支數，不得改寫已完成卡號。
- 每筆卡號列保存來源模式、機台、參考支數、載具、實秤毛重、皮重快照、淨重、秤重者、秤重時間及鎖定狀態。

### 7.2 進料載具

進料載具來源為客戶批號既有載具設定，分配到機台時保存：

- 來源訂單細項載具ID。
- 載具種類與所有權。
- 單個重量。
- 分配數量。
- 分配總重量。
- 對應卡號或生產紀錄。
- 後續處置狀態。

同一進料載具數量不得重複分配到多台機台。

### 7.3 良品出料載具

每台機台完成時，良品出料載具可採：

- `reused`：沿用進料載具。
- `replacement`：更換其他載具。
- 混合使用：同一結果同時存在沿用及更換明細，由資料自動判定。

沿用進料載具時：

- 出料載具明細必須指向來源進料載具。
- 沿用數量不得超過來源可用數量。
- 不得重複增加實體載具數量。

更換載具時：

- 新增實際出料載具種類、數量、單重及總重。
- 出料載具可不具有來源進料載具ID。
- 原進料載具必須記錄後續處置。

### 7.4 原進料載具後續處置

可用狀態：

- `reused_for_good`：沿用裝載良品。
- `return_empty`：空載具待歸還客戶。
- `stored_on_site`：空載具暫存廠內。
- `damaged`：載具損壞。
- `other`：其他，備註必填。

良品庫存只綁定實際出料載具，不可因換載具而改寫原始進料載具及進料重量。

### 7.5 重量區分

系統必須分開顯示：

- 進料內容物淨重。
- 進料載具重量。
- 進料毛重。
- 良品內容物淨重。
- 良品出料載具重量。
- 良品出料毛重。

## 8. 不良品塑膠袋

### 8.1 原則

- 不良品通常使用塑膠袋。
- 包裝單位可為「包」或「袋」。
- 塑膠袋不是載具，不使用載具主檔。
- 塑膠袋不計重量、不保存袋重、不列入載具總重。
- 不良品實秤重量直接作為內容物淨重。

### 8.2 必存資料

- 來源機台完成結果。
- 包裝編號。
- 包裝類型，預設為塑膠袋。
- 包裝單位：包或袋。
- 包裝數量。
- 所含不良支數。
- 不良品內容物重量。
- 包裝狀態。
- 備註。

### 8.3 分袋與出貨

若每袋分開秤量，可一袋一筆：

```text
袋01：50支
袋02：49支
合計：99支／2袋
```

若多袋合併秤量，可保存一筆合併包裝；合併包裝拆分前不得只出其中一袋，避免無法確定應扣除的支數。需要部分出貨時，應先執行拆包／重新包裝並保存異動紀錄。

## 9. 階段輸出與統計

### 9.1 終點與中間流

- 一般篩分直接完成的良品／不良品是終點輸出，可建立庫存。
- 由一般篩分轉入二次篩分的輸出是中間流，不建立庫存。
- 二次篩分完成後的良品／不良品是終點輸出，建立庫存。
- 以第二道工序進入二次篩分的來源仍是良品中間流，不得因「二次篩分」名稱而改列為不良品。
- 以放寬標準進入二次篩分的來源是不良品中間流，轉入數量使用秤重換算值。

### 9.2 避免重複計算

統計與庫存只能加總終點輸出：

```text
最終良品 = 一般篩分直接完成良品 + 二次篩分完成良品
最終不良品 = 一般篩分直接完成不良品 + 二次篩分完成不良品
```

已由一般篩分轉入二次篩分的數量不得再次列入終點庫存。

### 9.3 製程統計與庫存數量

- 良品終點輸出使用經確認的製程良品支數入庫。
- 不良品終點輸出使用秤重換算支數入庫。
- 機台原始不良支數仍保留於製程統計。
- 製程與庫存的差異不得偷偷調整良品；必須以差異欄位及原因呈現。
- 二次篩分不論來源方式，都會產生新的良品與不良品；其新不良品必須重新保存該次機台原始數量及實秤換算數量，圖片則依該二次篩分實例的圖片要求快照處理。

## 10. 良品與不良品庫存

### 10.1 庫存分類

`inventory_items` 新增不可混淆的 `stock_category`：

- `good`：良品。
- `defect`：不良品。

`stock_category` 表示庫存實體類別；`quality_status` 表示品質或可出貨狀態，兩者不可互相取代。

### 10.2 良品庫存

```text
stock_category = good
quality_status = qualified
total_good_units = 良品支數
total_defect_units = 0
quantity_on_hand = 良品支數
net_weight_kg = 良品內容物淨重
tool_weight_kg = 實際出料載具重量
gross_weight_kg = 良品淨重 + 出料載具重量
```

良品庫存必須關聯實際出料載具明細。

### 10.3 不良品庫存

```text
stock_category = defect
quality_status = rejected
total_good_units = 0
total_defect_units = 秤重換算不良支數
quantity_on_hand = 秤重換算不良支數
net_weight_kg = 不良品實秤重量
tool_weight_kg = 0
gross_weight_kg = 不良品實秤重量
```

不良品庫存關聯塑膠袋包裝明細，不關聯載具重量。

### 10.4 為何分成兩筆庫存

- 良品與不良品有不同可出貨規則。
- 兩者都有獨立現有量、保留量及已出貨量。
- 不良品需要依實際99支扣帳，不能只放在良品庫存的統計欄位。
- 出貨單可在同一張單加入兩筆庫存，分別扣帳。

### 10.5 庫存來源

每筆庫存都必須可追溯到：

- 訂單及訂單細項。
- 生產工單。
- 篩分階段。
- 機台執行。
- 機台完成結果。
- 階段輸出／轉流。
- 已上傳的原始圖片與雙數量；圖片選填且未上傳時仍須保留完整結構化數量。

## 11. 出貨與客戶歸還

### 11.1 出貨性質

| 出貨性質 | 允許內容 |
|---|---|
| `normal` 一般出貨 | 良品庫存 |
| `defect_return` 不良回送 | 不良品庫存 |
| `mixed` 混合出貨 | 良品庫存、不良品庫存及載具 |
| `tool_return` 載具歸還 | 空載具 |

### 11.2 不良品出貨

- 出貨庫存扣除基準為入庫不良支數，例如99支。
- 出貨明細同時顯示支數與包裝，例如 `99支／2袋`。
- 出貨時必須選擇實際袋號或包裝明細。
- 出貨一袋時，扣除該袋所含支數；不得只扣袋數。
- 撤銷出貨時，支數及包裝狀態必須一起恢復。

### 11.3 良品及載具出貨

- 出貨良品時帶入其實際出料載具。
- 更換載具的工單不得誤帶原進料載具。
- 待歸還空載具可在相同混合出貨單或獨立載具歸還單處理。

### 11.4 不良品摘要

`shipping_order_defect_summaries` 改為由實際加入出貨單的不良品庫存自動彙總，不再作為獨立手工庫存來源。摘要可保留作報表快照，但不能取代庫存扣帳。

## 12. Modal最終版面

### 12.1 共用原則

- 新增與編輯共用同一套階段及機台元件。
- 新增Modal可由兩個入口開啟：確認單「轉為工單」開啟時直接帶入來源訂單細項；工單頁面「新增工單」開啟時先讓使用者選擇來源確認單及訂單細項。
- 來源訂單細項確定後，兩個入口顯示相同欄位、頁籤、預帶資料及驗證結果；「轉為工單」不得維護另一份Modal或另一套欄位契約。
- 新增模式開放工單與機台規劃；實際檢驗、完成、入庫與出貨需在工單建立後執行。
- 沿用既有 `.work-order-screening-stage-tabs`、`.split-work-order-layout`、`.split-machine-tabs`、`form-grid`、`inline-label`、`ui-compact-form-row`、`.op-action-btn` 及既有 `data-action` 契約。
- 不新增局部仿製CSS、固定px、inline style或近似共用元件。

### 12.2 主頁籤

```text
工單設定｜生產與篩分｜二次篩分｜庫存與結案｜交辦與備註
```

- 一般流程固定顯示「生產與篩分」，不標示「第一階段」。
- 尚未產生二次篩分轉流時，不顯示「二次篩分」頁籤。
- 一旦建立二次篩分，頁籤固定保留以供後續追溯，即使流程已完成或取消也不隱藏。
- 不再顯示一般／拆分工單切換。

### 12.3 生產與篩分／二次篩分內容

一般流程與二次篩分都使用既有機台清單：

```text
機台A｜機台B｜＋加開機台
```

右側機台內容：

1. 任務與排程。
2. 卡號與進料載具。
3. 首件與篩分檢驗。
4. 完成確認。
5. 輸出與轉流。

### 12.4 完成確認區

固定顯示：

```text
機台顯示良品：900支
機台顯示不良：100支
不良品實秤：1.000kg
單支重快照：10.1g
入庫不良：99支
差異：-1支
不良包裝：2袋
機台畫面：已上傳1張
```

「機台畫面」同時顯示該工單的要求狀態：

- `選填｜未上傳`：可正常完成。
- `選填｜已上傳1張`：可正常完成並保留圖片。
- `必填｜尚缺1張`：禁止完成。
- `必填｜已符合`：可正常完成。

良品區另顯示：

- 進料載具。
- 出料載具方式：沿用／更換／混合。
- 實際出料載具。
- 待處理空載具。

卡號與進料載具區保留兩種模式：

```text
生管預設｜自行輸入
```

- 生管預設：依訂單載具快照建立載具列並自動計算卡號。
- 自行輸入：由現場新增、刪除或調整載具列，卡號仍由系統自動計算。
- 每列固定顯示「卡號（累計支數）／載具／載具皮重／實秤毛重／內容物淨重／秤重時間」。
- 已填實秤毛重的列顯示鎖定狀態；新增或刪除其他列時不得重算該列。

### 12.5 輸出與轉流區

完成後顯示兩張輸出卡：

- 良品輸出卡：直接入庫，或以「第二道工序」方式進入二次篩分。
- 不良品輸出卡：直接不良入庫，或以「放寬標準」方式進入二次篩分。

卡片必須顯示來源階段、機台、數量、重量、載具／包裝及目前處置狀態。

若同時存在兩種二次篩分方式，「二次篩分」頁面以方式切換：

```text
第二道工序（來源：一般篩分良品）｜放寬標準（來源：一般篩分不良品）
```

兩種方式各自顯示自己的來源、標準、機台及結果。

- 第二道工序畫面不得顯示「不良來源」或強制要求放寬佐證。
- 放寬標準畫面必須顯示機台原始不良支數、秤重換算支數、差異及放寬佐證。
- 二次篩分機台完成後，兩種方式都使用相同的良品／不良品結果確認元件。

## 14. API與交易邊界

### 14.1 API職責

建議拆出明確端點：

- 共用工單建立端點：同時接受確認單轉入與工單頁面直接新增，並對來源訂單細項執行相同驗證及欄位映射。
- 工單階段規劃。
- 機台新增、更新、取消。
- 首件檢驗新增及判定。
- 機台完成結果草稿。
- 機台圖片上傳。
- 機台結果確認。
- 階段輸出處置及轉流。
- 終點輸出入庫。
- 結果撤銷／更正。

現有 `api/work_orders/update.php` 不應繼續承擔所有結果、轉流及入庫副作用。

### 14.2 機台結果確認交易

單一資料庫交易內依序：

1. `SELECT ... FOR UPDATE` 鎖定工單、階段及機台。
2. 驗證狀態、首件、服務明細、載具，以及圖片要求快照中列為必填的圖片。
3. 後端依實秤重量及單支重計算入庫不良支數。
4. 保存原始數量、換算數量、差異及公式快照。
5. 保存良品出料載具及不良品包裝。
6. 建立良品／不良品階段輸出。
7. 寫入操作與稽核紀錄。
8. 提交交易。

### 14.3 轉流與入庫交易

- 鎖定來源結果及既有轉流。
- 驗證可用輸出數量。
- 中間流建立目標階段來源，不建立庫存。
- 終點流分別建立良品或不良品庫存、來源、載具／包裝及庫存交易。
- 使用唯一鍵或冪等識別避免重複入庫。

### 14.4 禁止刪除重建機台

既有 `replaceWorkOrderMachineRuns()` 的刪除全部再重建方式必須退出新流程，改為依穩定ID逐筆：

- 新增。
- 更新。
- 取消。
- 保留歷史。

已有結果、轉流、入庫或出貨的機台不可實體刪除。

## 15. 驗證與守門

### 15.1 建立與規劃階段

- 確認單「轉為工單」與工單頁面「新增工單」必須使用同一建立端點及交易。
- 兩個入口都必須綁定有效來源訂單細項，且不得繞過來源狀態、可轉數量或重複建立守門。
- 相同來源訂單細項由不同入口建立時，預帶的客戶、品項、數量、服務規格、交期及載具必須一致。
- 入口資訊不得作為工單類型、流程分支或報表分類條件。
- 一般生產與篩分流程至少一台機台。
- 同一機台不可在時間重疊且不允許重複排程的情況下重複配置。
- 各機台分配重量及支數不得超過工單可分配量。
- 進料載具分配不得重複使用。

### 15.2 開始生產

- 機台、負責人及必要排程完整。
- 單支重大於0。
- 首件尺寸已通過。
- 卡號與進料載具規劃完整。

### 15.3 機台完成

- 原始良品、不良品及處理總數完整。
- 篩分服務不良明細完整。
- 不良品實秤重量合法。
- 圖片要求為選填時，缺圖可正常完成，不需要豁免。
- 圖片要求為必填時，對應圖片類型及最低張數必須完整。
- 良品出料載具完整。
- 不良品大於0時，包／袋資料完整。
- 後端換算結果與前端預覽一致。

### 15.4 轉流

- 轉流數量不得大於來源可用數量。
- 同一來源輸出不可重複入庫。
- 二次篩分必須指定方式，且只能是第二道工序或放寬標準。
- 第二道工序方式的來源品質必須是good，來源數量必須取自一般篩分良品，且不得帶入不良歷史來源。
- 放寬標準方式的來源品質必須是defect，來源數量必須取自秤重換算不良支數。
- 放寬標準方式必須有放寬後規格與佐證。
- 第二道工序與放寬標準不得共用同一執行實例的規格及結果。
- 轉入二次篩分的不良品使用入庫換算支數，不使用機台原始100支。

### 15.5 出貨

- 一般出貨只允許良品。
- 不良回送只允許不良品。
- 混合出貨允許良品、不良品及載具。
- 不良品出貨必須選擇實際包裝。
- 已出貨或已分配庫存不得直接撤銷上游機台結果。

## 16. 舊資料與相容策略

1. 為既有一般與拆分工單補建內部預設篩分階段，介面名稱仍顯示「生產與篩分」。
2. 既有拆分機台全部掛入預設篩分階段。
3. 既有一般工單若只有主表機台，建立一筆預設篩分機台執行。
4. 既有 `production_records.machine_run_id` 有值者直接保留關聯。
5. 單機工單的舊首件尺寸可自動綁定第一台機台。
6. 多機但無法判斷歸屬的舊首件尺寸保留為歷史未分配，不得任意指定。
7. 既有機台不良明細保留原始數量語意。
8. 既有完成工單及庫存不重新計算100／99，不改寫歷史庫存。
9. 既有庫存預設回填 `stock_category = good`；只有明確的不良庫存才可回填為defect。
10. `work_order_type` 暫時保留供舊API與報表相容，新畫面不再讓使用者切換。

## 17. 現有二次篩分模組邊界

### 工單內二次篩分

- 尚未形成正式庫存。
- 第二道工序方式的來源是同工單一般篩分良品。
- 放寬標準方式的來源是同工單一般篩分不良品。
- 使用 `work_order_stages` 與 `work_order_stage_transfers`。

### 現有 `rescreen_batches`

- 來源通常已入庫、已出貨、客退或既有不良歷史。
- 可建立獨立重篩案件及重篩工單。
- 保留處理工單外或庫存後的重篩。

不能只因名稱都有「二次篩分」就將第二道工序型良品送入既有不良重篩案件。兩者可共享換算、卡號、圖片及服務驗證函式，但不可共用錯誤的來源品質與生命週期語意。

## 21. 驗收案例

### Case A：單機、不需二次篩分

1. 一般生產與篩分流程只有機台A。
2. 機台顯示良品900、不良100。
3. 不良品實秤換算為99，包裝2袋。
4. 良品沿用進料載具。
5. 直接建立：
   - 良品庫存900支。
   - 不良品庫存99支／2袋。
6. 可由工單反查100、99、差異及圖片。

### Case B：趕工、同階段兩台機台

1. 在一般生產與篩分流程加開機台B。
2. 機台A與B各有獨立卡號、載具、首件、重量及不良品。
3. 階段統計正確彙總兩台機台。
4. 任一機台未完成時，階段不得誤判完成。

### Case C：良品更換載具

1. 進料為鐵桶10桶。
2. 良品改裝塑膠籃20籃。
3. 良品庫存只顯示塑膠籃20籃。
4. 原鐵桶10桶顯示待歸還或其他處置。
5. 進料與出料重量互不覆蓋。

### Case D：兩種二次篩分方式

1. 一般篩分良品以「第二道工序」方式轉入二次篩分。
2. 該來源仍標示為良品，不建立不良歷史，不要求放寬佐證。
3. 一般篩分不良品99支以「放寬標準」方式轉入二次篩分。
4. 不良來源使用99支，不使用機台原始100支，並要求放寬佐證。
5. 一般篩分轉流數量不建立庫存。
6. 系統建立兩個不同的二次篩分執行實例。
7. 兩個實例各自保存來源品質、標準、機台及結果。
8. 兩種方式的二次篩分都重新產生良品與不良品。
9. 只有二次篩分終點良品／不良品建立庫存。

### Case E：混合出貨

1. 同一出貨單加入良品庫存及不良品庫存。
2. 不良品選擇實際2袋。
3. 出貨完成後扣除正確良品支數、不良品99支及2袋。
4. 同單可歸還更換後留下的空載具。
5. 撤銷出貨時三者完整恢復。

### Case F：防重與不可改寫

1. 已確認機台結果不可直接改寫100、99或圖片。
2. 同一輸出不可重複入庫。
3. 已出貨庫存不可撤銷上游結果。
4. 單支重主檔修改後，歷史99不變。
5. 放寬標準不覆蓋一般篩分原始規格。

### Case G：兩種工單建立入口

1. 從確認單的某筆訂單細項按「轉為工單」，新增Modal直接帶入該筆來源資料。
2. 從生產工單頁面按「新增工單」，選取同一筆確認單及訂單細項。
3. 兩個入口帶入的客戶、品項、數量、服務規格、交期及載具資料一致。
4. 兩個入口送出相同資料時，呼叫同一建立交易並產生相同工單、一般篩分階段及第一台機台結構。
5. 已被有效工單使用或已無可轉數量的訂單細項，兩個入口都受到相同阻擋。
6. 建立入口不影響工單類型、後續製程、庫存、報表或編輯Modal。

### Case H：依客戶要求決定圖片是否必填

1. 客戶甲的圖片要求預設為選填；建立工單後保存 `optional` 快照。
2. 客戶甲的機台未上傳圖片，仍可正常確認完成、入庫及結案，不需要缺圖豁免。
3. 客戶乙的圖片要求預設為必填；建立工單後保存 `required` 及最低張數快照。
4. 客戶乙未達圖片最低張數時，後端阻擋機台完成；補齊後才允許完成。
5. 某筆訂單細項明確覆寫客戶預設時，只影響由該細項建立的工單。
6. 工單建立後修改客戶預設，不得改變既有工單的圖片要求快照。
7. 二次篩分沿用工單快照；若有明確階段覆寫，該實例依自己的快照驗證。

## 23. 建議測試重點

- 確認單轉工單與工單頁面直接新增的預帶資料、驗證錯誤及建立結果一致。
- 同一訂單細項由兩個入口同時送出時，重複建立與可轉數量守門仍有效。
- 四捨五入邊界：98.49、98.50、99.49、99.50。
- 單支重為0、負數或缺失。
- 機台原始數量與服務明細合計不一致。
- 不良品有數量但沒有實秤或袋數；圖片缺失則分別驗證選填可完成、必填被阻擋。
- 客戶圖片預設為選填／必填、訂單細項覆寫，以及建立工單後修改客戶設定。
- 多機台同時完成時的鎖定與防重。
- 同一進料載具重複分配。
- 沿用載具數量超過來源。
- 更換載具後原載具未處置。
- 一般篩分輸出部分直接入庫、部分轉入二次篩分。
- 二次篩分來源同時包含良品與不良品。
- 同一工單同時存在第二道工序與放寬標準兩種二次篩分方式。
- 重複送出完成、轉流、入庫或出貨請求。
- 不良品合併包裝嘗試部分出貨。
- 出貨後撤銷、退回、再出貨。
- 歷史資料無法判斷機台歸屬時不任意回填。
