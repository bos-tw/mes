# Apache 模組基線清單（MES）

建立日期：2026-05-10
適用系統：Windows Apache（本專案路徑 C:\Apache24\htdocs\mes）

## 1. 目的

此文件用於統一測試環境與正式環境的 Apache 模組配置，避免因模組差異導致：

- 500 Internal Server Error（例如 Invalid command 'Header'）
- 上傳目錄保護規則失效
- 入口頁或路由行為不一致

## 2. 必開模組（Required）

以下模組為本專案目前規則直接使用到，建議測試機與正式機都開啟：

1. headers_module
   - 用途：支援 Header 指令（快取控制）。
   - 關聯檔案：/.htaccess。

2. authz_core_module
   - 用途：支援 Require all denied / granted（存取控制）。
   - 關聯檔案：/uploads/.htaccess。

3. dir_module
   - 用途：支援 DirectoryIndex（入口頁優先順序）。
   - 關聯檔案：/.htaccess。

## 3. 建議模組（Recommended）

1. rewrite_module
   - 目的：支援 URL rewrite 與後續擴充需求。

2. setenvif_module
   - 目的：常見條件式設定與相容性需求。

3. ssl_module（正式站使用 HTTPS 時）
   - 目的：TLS 憑證與 HTTPS。

4. deflate_module、expires_module（效能優化）
   - 目的：壓縮與快取策略管理。
   - 備註：若內網系統以即時更新為主，可視需求再啟用。

## 4. httpd.conf 設定範例

在 Apache 設定中確認以下 LoadModule 行存在（已存在則不用重複新增）：

```apache
LoadModule headers_module modules/mod_headers.so
LoadModule authz_core_module modules/mod_authz_core.so
LoadModule dir_module modules/mod_dir.so
LoadModule rewrite_module modules/mod_rewrite.so
LoadModule setenvif_module modules/mod_setenvif.so
# 若使用 HTTPS
# LoadModule ssl_module modules/mod_ssl.so
# 效能優化（選配）
# LoadModule deflate_module modules/mod_deflate.so
# LoadModule expires_module modules/mod_expires.so
```

## 5. 上線前驗收步驟

### 步驟 A：檢查模組是否載入

```powershell
C:\Apache24\bin\httpd.exe -M | Select-String "headers|authz_core|dir|rewrite|setenvif|ssl|deflate|expires"
```

### 步驟 B：重啟 Apache

依你的服務管理方式重啟 Apache（例如 Windows 服務或 ApacheMonitor）。

### 步驟 C：驗證專案可用性

```powershell
Invoke-WebRequest http://localhost/mes/login.html -SkipHttpErrorCheck
Invoke-WebRequest http://localhost/mes/api/session.php -SkipHttpErrorCheck
Get-Content -Path "C:\Apache24\logs\error.log" -Tail 80
```

預期：

- login.html 回應 200
- api/session.php 在未登入時回應 401（屬正常）
- error.log 不應再出現 Invalid command 'Header'

## 6. 測試機/正式機一致性檢查表

1. Apache 主版本一致（建議）。
2. 必開模組一致（headers/authz_core/dir）。
3. 建議模組一致（rewrite/setenvif/ssl，依需求）。
4. /uploads/.htaccess 存取規則有效。
5. /mes/login.html 與主要 API 回應碼一致。
6. error.log 無致命設定錯誤。

## 7. 已採用的防呆措施

專案目前已在 /.htaccess 將 Header 規則包在 IfModule mod_headers.c，
即使某環境未啟用 headers_module，也不會直接造成 500。

注意：未啟用 headers_module 仍會失去 no-cache 標頭，建議正式環境維持啟用。
