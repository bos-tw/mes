## 🧪 測試步驟（每個模組）

### 1️⃣ 開啟模組
- 登入系統
- 導航至目標模組
- **檢查點 1**：頁面正常載入，無 Console 錯誤

### 2️⃣ 測試「新增」功能
- 點擊「新增」按鈕
- **檢查點 2**：Modal 正常開啟
- **檢查點 3**：Console 無 `欄位不存在` 警告
- **檢查點 4**：所有欄位可正常輸入
- 填入測試資料並儲存
- **檢查點 5**：資料成功儲存

### 3️⃣ 測試「編輯」功能
- 點擊任一筆資料的「編輯」按鈕
- **檢查點 6**：Modal 正常開啟
- **檢查點 7**：Console 無 `欄位不存在` 警告
- **檢查點 8**：所有欄位已正確填入原始資料
- 修改部分欄位並儲存
- **檢查點 9**：資料成功更新

### 4️⃣ Console 檢查
- 開啟瀏覽器開發者工具（F12）
- 切換到 Console 分頁
- **檢查點 10**：無紅色錯誤訊息
- **檢查點 11**：無 `Cannot read properties of null` 錯誤
- **檢查點 12**：無 `欄位不存在` 警告（除非真的缺欄位）

## 🚨 常見問題排查

### 問題 1：Console 出現 `欄位不存在 - id`
**原因**：配置檔缺少 `hiddenFields: ['id']`
**解決**：檢查 `core/configs/{模組名稱}.config.js`，在 modal 中加入 `hiddenFields: ['id']`

### 問題 2：編輯時某些欄位沒有填入資料
**原因**：JS 中該欄位仍使用舊的 `querySelector().value =` 方式
**解決**：檢查 `js/{模組名稱}.js`，確認所有欄位都使用 `setFieldValue()`

### 問題 3：Console 出現 `Cannot read properties of null (reading 'value')`
**原因**：某個 querySelector 未使用防禦性函數
**解決**：找到錯誤行號，改用 `setFieldValue()` 或加入 null 檢查
