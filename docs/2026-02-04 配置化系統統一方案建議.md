## 🔍 問題根源分析

### 為什麼配置化後出錯？

**核心問題**：JS 程式碼與配置化 Modal 結構不匹配

#### 範例：dashboard_calendar_events

**JS 期望的 HTML**：
```html
<form data-dashboard-calendar-events-form>
    <input type="text" name="title">
    <input type="datetime-local" name="start_datetime">
    ...
</form>
```

**配置化產生的結構**：
```html
<form data-dashboard-calendar-events-form>
    <section class="form-section">
        <h4>基本資訊</h4>
        <div class="form-grid">
            <label class="inline-label">
                <span>標題</span>
                <input type="text" name="title">
            </label>
            ...
        </div>
    </section>
</form>
```

**問題**：
- ❌ `querySelector('input[name="title"]')` 仍能找到
- ❌ 但如果配置檔設定錯誤（缺少某些欄位），querySelector 會返回 null
- ❌ JS 沒有做 null 檢查 → 出錯

## 💡 統一方案建議

### 方案 A：全面配置化（推薦）⭐

**目標**：所有模組都使用配置化，徹底統一標準

#### 執行步驟

**1️⃣ 修正配置檔缺失欄位**

問題模組的配置檔可能缺少某些欄位定義，導致 Modal 無法正確渲染。

```javascript
// 範例：確保所有 JS 使用的欄位都在配置中定義
modal: {
    sections: [
        {
            title: '基本資訊',
            fields: [
                { name: 'title', type: 'text', required: true },
                { name: 'start_datetime', type: 'datetime-local', required: true },
                // ⚠️ 確保所有欄位都有定義
            ]
        }
    ]
}
```

**2️⃣ 修正 JS 程式碼加入防禦性檢查**

```javascript
// 修正前（會出錯）
modalForm.querySelector('input[name="title"]').value = data.title;

// 修正後（加入 null 檢查）
const titleInput = modalForm.querySelector('input[name="title"]');
if (titleInput) {
    titleInput.value = data.title || '';
} else {
    console.warn('找不到 title 欄位');
}
```

**3️⃣ 特殊模組使用混合模式**

對於極複雜的模組（如 dashboard），使用 `requiresHtmlModal: true`：

```javascript
ModuleConfig.register('dashboard', {
    title: '系統儀表板',
    requiresHtmlModal: true,  // Modal 從 HTML 載入
    customHtml: `
        <!-- 完整的自訂 HTML 結構 -->
        <section class="dashboard-section">...</section>
    `
});
```

#### 優點
- ✅ **統一標準**：所有模組都在配置系統中
- ✅ **易於維護**：修改 Modal 樣式只需改配置檔
- ✅ **結構一致**：自動套用系統風格
- ✅ **可追蹤**：所有模組都註冊在 ModuleConfig

#### 缺點
- ⚠️ 需要修正 3-8 個模組的 JS 程式碼
- ⚠️ 需要檢查配置檔完整性
- ⚠️ 一次性工作量較大

---

### 方案 B：全面退回傳統 HTML

**目標**：放棄配置化，所有模組都用 HTML

#### 執行步驟

1. 刪除所有 `.config.js` 檔案
2. 恢復所有 `.html.bak` 為 `.html`
3. 移除 ModuleConfig 和 ModuleRenderer
4. script.js 只載入 HTML

#### 優點
- ✅ JS 程式碼完全不需改
- ✅ 結構清楚，一目了然

#### 缺點
- ❌ **大量重複代碼**：每個 Modal 都要寫一遍 HTML
- ❌ **維護困難**：改 Modal 樣式要改 39 個檔案
- ❌ **違反 DRY 原則**：完全放棄統一化的好處
- ❌ **浪費之前的配置化努力**

---

### 方案 C：混合標準（不推薦）

**目標**：標準 CRUD 用配置，複雜模組用 HTML

#### 問題
- ❌ **兩套標準**：開發者需要記住哪些用哪種方式
- ❌ **混亂**：正是現在的狀況
- ❌ **違反統一化目標**

---

## 🎯 最終建議

### 推薦：**方案 A - 全面配置化** ⭐⭐⭐⭐⭐

#### 為什麼？

1. **符合原始目標**：當初就是要「統一化」
2. **長期維護性好**：配置檔比 HTML 更易維護
3. **問題可解決**：只需修正 JS 和配置檔
4. **保留彈性**：可用 customHtml 或 requiresHtmlModal 處理特殊情況

#### 執行計畫

```javascript

modal: {
    sections: [
        {
            title: '基本資訊',
            fields: [
                { name: 'title', type: 'text', required: true },
                { name: 'event_type', type: 'select', required: true, options: [...] },
                { name: 'status', type: 'select', options: [...] },
                { name: 'priority', type: 'select', options: [...] },
                // ✅ 確保所有欄位都在
            ]
        },
        {
            title: '時間設定',
            fields: [
                { name: 'start_datetime', type: 'datetime-local', required: true },
                { name: 'end_datetime', type: 'datetime-local' },
                { name: 'is_all_day', type: 'checkbox', label: '全天事件' },
                { name: 'color', type: 'color', defaultValue: '#3788d8' },
            ]
        },
        {
            title: '說明',
            fields: [
                { name: 'description', type: 'textarea', fullWidth: true, rows: 4 }
            ]
        }
    ]
}
```

##### 階段 3：修正 JS 程式碼（2-4 小時）

為所有 querySelector 加入防禦性檢查：

```javascript
function openModal(mode, data = null) {
    // 清空表單
    modalForm.reset();

    // 安全的欄位設定函數
    function setFieldValue(name, value) {
        const field = modalForm.querySelector(`[name="${name}"]`);
        if (field) {
            if (field.type === 'checkbox') {
                field.checked = !!value;
            } else {
                field.value = value || '';
            }
        } else {
            console.warn(`欄位不存在: ${name}`);
        }
    }

    if (mode === 'edit' && data) {
        setFieldValue('title', data.title);
        setFieldValue('event_type', data.event_type);
        setFieldValue('start_datetime', data.start_datetime);
        setFieldValue('end_datetime', data.end_datetime);
        setFieldValue('is_all_day', data.is_all_day);
        setFieldValue('color', data.color);
        setFieldValue('description', data.description);
    }

    modal.classList.remove('hidden');
}
```

ModuleConfig.register('dashboard', {
