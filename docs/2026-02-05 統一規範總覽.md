# 統一規範總覽

> 📅 建立日期：2026-02-05
> 🎯 目的：建立一致的程式碼、UI 與配置規範，避免重複修改與風格分裂

---

## 1) 統一架構原則

### 1.1 配置化優先
- 有配置檔就使用 ModuleRenderer
- 無配置檔才載入 modules/*.html
- 需要混合模式（複雜 Modal）才使用 requiresHtmlModal

### 1.2 統一配置規格
- 所有 modal 必須包含 `hiddenFields: ['id']`
- filters / columns / actions 必須使用統一格式
- 禁止使用舊格式（headerActions / dataAction 等）

---

## 2) JS 程式碼一致性

### 2.1 防禦性賦值（必須）
```javascript
function setFieldValue(name, value) {
    const field = form.querySelector(`[name="${name}"]`);
    if (field) {
        if (field.type === 'checkbox') {
            field.checked = !!value;
        } else {
            field.value = value || '';
        }
    } else {
        console.warn(`模組: 欄位不存在 - ${name}`);
    }
}
```

### 2.2 統一事件綁定方式
- moduleRoot 上集中使用事件委派
- modal 與 detail modal 分開處理
- 禁止直接綁定不存在元素

---

## 3) UI 統一規範（摘要）

### 3.1 按鈕
- 主操作：btn primary
- 次要：btn outline
- 表格操作：btn text（danger 只限刪除）

### 3.2 Modal
- 尺寸只允許：xlarge / large / medium / small
- 結構必須使用 modal-overlay / modal-window / modal-close

### 3.3 表單
- label 統一 inline-label
- 全寬欄位使用 full-width

---

## 4) 資料一致性規範

- 所有 CRUD 必須檢查關聯完整性
- 確保 API 回傳結構與前端表格欄位一致
- 避免在 API 中混用欄位名稱

---

## 5) 驗證與品質門檻

### 5.1 配置驗證（必須）
- 修改配置前後必須執行：
  - node tools/validate-config-modules.js

### 5.2 檢查門檻
- 驗證工具 **零錯誤** 才能視為完成
- JS 防禦性函數不得缺漏
- 混合模式 HTML 必須存在
