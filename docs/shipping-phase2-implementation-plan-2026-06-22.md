# 出貨單第二階段實作規劃

更新日期：2026-06-22  
狀態：待實作  
前置條件：第一階段 P0 已完成，進入實機驗證與第二階段規劃  
適用範圍：出貨品質檢驗、不良品追溯 / 回送、客戶載具往來總帳、退貨後二次重篩管制、跨模組追溯鏈

## 1. 第二階段目標

第一階段已完成可運作的出貨單骨幹，但仍以「摘要可用、流程可跑」為主。

第二階段目標不是重做第一階段，而是在不破壞現場已可使用流程的前提下，補齊下列能力：

- 出貨品質檢驗可被查詢、建立並與出貨單關聯。
- 不良品回送可追溯原工單、原庫存、原出貨與後續退貨重篩。
- 客戶載具不再只是一張出貨單上的快照，而是能形成往來總帳。
- 出貨、退貨、部分入庫、工單、檢驗之間可構成完整追溯鏈。

## 2. 第二階段完成定義

本階段最低可交付結果：

- 使用者可從出貨單直接看到或開啟對應的出貨品質檢驗。
- 不良品回送資料可以回推：
  - 原工單
  - 原庫存
  - 原出貨單
  - 後續退貨 / 二次重篩
- 客戶載具可查詢「帶入、歸還、未結清」狀態。
- 出貨詳情頁可顯示主要來源與後續流向。
- 第二階段新增能力不強迫改寫第一階段既有資料輸入方式，需與既有測試流程相容。

## 3. 範圍

### 3.1 本階段會做

- 出貨品質檢驗與出貨單關聯
- 不良品追溯與回送管制
- 客戶載具往來總帳
- 退貨後二次重篩來源鏈規劃
- 出貨詳情跨模組追溯鏈顯示

### 3.2 本階段先不做滿

- 全新的出貨紙本樣式家族
- 以財務結算為目的的載具應收應付
- 不良品完整成本分攤
- 自動派生二次重篩工單的全閉環流程

## 4. 主題一：出貨品質檢驗整合

### 4.1 背景

目前實務上尚未全面執行出貨品質檢驗，但系統上已需要預留，以便未來從「可選」升級為「應做」甚至「必做」。

### 4.2 第二階段目標

- 出貨單可顯示是否已有品質檢驗。
- 使用者可從出貨單直接建立或開啟品質檢驗。
- 檢驗結果可作為後續守門依據，但第二階段初期先不強制卡流程。

### 4.3 建議實作

- 在出貨單詳情區加入：
  - 檢驗筆數
  - 最近檢驗日期
  - 檢驗狀態
- 在出貨單操作列或詳情中加入「品質檢驗」入口。
- 建立出貨單與 `shipping_quality_inspections` 的關聯欄位或查詢口徑。

## 5. 主題二：不良品追溯與回送

### 5.1 背景

不良品會隨出貨單回送客戶，之後可能再退回並進行二次重篩，因此不能只停在第一階段的摘要記錄。

### 5.2 第二階段目標

- 每筆出貨不良品摘要能回推來源工單與來源庫存。
- 若後續發生退貨與重篩，可追到第一次出貨單。
- 不良品歷史查詢可跨工單、出貨、退貨、重篩瀏覽。

### 5.3 建議實作

- 明確定義不良品摘要來源欄位的必填 / 選填規則。
- 在不良品歷史查詢頁加入：
  - 來源出貨單
  - 來源庫存
  - 退貨單
  - 二次重篩標記
- 規劃退貨後二次重篩的來源控制欄位，例如：
  - `source_shipping_order_id`
  - `source_return_order_id`
  - `rescreen_work_order_id`

## 6. 主題三：客戶載具往來總帳

### 6.1 背景

第一階段只記錄「本次出貨隨貨歸還了哪些載具」。第二階段要處理的是「還剩哪些載具未歸還、何時帶入、何時歸還」。

### 6.2 第二階段目標

- 釐清客戶載具的流入與流出來源。
- 可查詢某客戶目前載具欠還狀態。
- 出貨摘要與載具總帳之間能互相回推。

### 6.3 建議實作

- 建立 `customer_tool_ledger` 或等價資料模型。
- 將下列事件視為總帳來源：
  - 工單 / 部分入庫帶入載具
  - 出貨單歸還載具
  - 退貨單返還或再入場載具
- 在總帳中保留：
  - 客戶
  - 載具名稱 / 類型
  - 數量
  - 單重 / 總重
  - 來源模組 / 單號
  - 方向（in / out）
  - 結餘

## 7. 主題四：退貨後二次重篩管制

### 7.1 背景

退貨產品會進行二次重篩，因此退貨單不應只是單獨事件，而應能回接第一次出貨及後續新工單。

### 7.2 第二階段目標

- 退貨單可明確指出原出貨單來源。
- 若進入二次重篩，可識別是「重篩流程」而非一般新訂單。
- 二次重篩結果可回推第一次出貨與退貨。

### 7.3 建議實作

- 評估是否建立「重篩工單」標記或工單類型。
- 在退貨與工單之間加入來源關聯欄位。
- 在出貨 / 退貨 / 工單詳情頁中顯示二次重篩追溯鏈。

## 8. 主題五：跨模組追溯鏈

第二階段詳情頁建議可顯示以下對應：

- 訂單
- 工單
- 部分入庫
- 庫存項目
- 出貨品質檢驗
- 退貨單
- 二次重篩工單
- 不良品歷史
- 客戶載具總帳

原則：

- 先以可查詢、可跳轉為主。
- 守門規則可晚於追溯畫面上線。

## 9. 第二階段開發順序建議

1. 先補出貨品質檢驗與出貨單顯示 / 入口關聯
2. 再補不良品追溯欄位與歷史查詢串接
3. 再建客戶載具往來總帳
4. 最後補退貨後二次重篩管制與更嚴格流程守門

## 10. 風險與注意事項

- 不可破壞第一階段已可運作的出貨流程。
- 第二階段若直接引入強制守門，容易卡住現場測試，因此應先顯示、後阻擋。
- 客戶載具總帳若一次做到最完整，範圍會快速擴大，需先定義最小可用結餘模型。
- 不良品回送與二次重篩若未定義來源欄位，後續會很難追。
