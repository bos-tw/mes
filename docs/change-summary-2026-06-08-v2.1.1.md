# v2.1.1 更新摘要（2026-06-08）

本更新包彙整本輪生產工單長對話期間的全部工作區異動，供遠端一鍵更新與整體 UI/流程回歸測試使用。

## 核心更新
- 一般工單編輯 modal 依稿件重整版面，包含上方摘要列、訂單詳細資訊、圖面附件、生產排程、篩分明細、首件尺寸檢驗與生產記錄。
- 右側生產統計面板調整為稿件指定欄位，移除差值面板，並同步一般/拆分工單的即時計算邏輯。
- 生產記錄新增載具種類、載具重量與預設/自行輸入雙模式，卡號依支數與載具列數自動分配。

## 拆分工單
- 拆分工單套用一般工單的 section 結構、白底卡片樣式與收合箭頭行為。
- 保留拆分機台頁籤與機台級輸入邏輯，拆分機台內支援各機台排程、篩分明細、生產設定與生產記錄。
- 移除拆分模式內重複的工單摘要與重複全域生產記錄，避免資訊重複。
- 拆分機台篩分服務明細修正欄位重疊，服務項目改為完整單行顯示，備註改為可輸入欄位。

## API / 前端修正
- 修正 `work_orders.js` 圖面附件流程缺少 `formatFileSize()` 導致編輯工單載入失敗。
- `api/work_orders/*` 配合一般/拆分工單 UI 與生產記錄資料結構調整查詢、儲存與刪除邏輯。
- `modules/work_orders.html`、`styles.css`、`js/work_orders.js` 同步整理，避免一般/拆分模式切換殘留舊版面。

## 資料庫變更
- 包含 migration：`migrations/2026_06_07_add_production_record_tool_fields.sql`
- 本機 schema 同步腳本：`tools/sync-local-schema.ps1` 已更新，會補齊生產記錄的載具相關欄位。
