# 退貨單追溯系統分析

## 一、資料庫關聯架構

### 完整的追溯鏈

```
委託確認單 (orders)
    ↓ order_id
出貨單 (shipping_orders)
    ↓ original_shipping_order_id
退貨單 (return_orders)
```

### 明細層級的追溯鏈

```
訂單品項 (order_items)
    ↓ order_item_id
出貨單品項 (shipping_order_items)
    ↓ order_item_id (直接連結)
退貨單品項 (return_order_items)
```

**關鍵發現**：退貨單明細 (`return_order_items`) 直接連結到訂單品項 (`order_items`)，而非出貨單品項！

## 二、資料表結構分析

### 2.1 orders (委託確認單)
```sql
主鍵：id (bigint)
欄位：
  - order_number (varchar 50) UNIQUE  -- 委託單號
  - customer_id (bigint)               -- 客戶ID
  - order_date (date) INDEX            -- 訂單日期
  - expected_delivery_date (date)      -- 預交日期
  - customer_po_number (varchar 100)   -- 客戶訂單號
  - status (varchar 50) INDEX          -- 狀態
  - status_lookup_id (bigint)          -- 狀態(關聯LookupValues)
  - total_amount (decimal 14,2)        -- 總金額
  - notes (text)                       -- 備註
```

### 2.2 shipping_orders (出貨單)
```sql
主鍵：id (bigint)
外鍵：
  - order_id → orders(id) ON DELETE SET NULL
  - customer_id → customers(id) ON DELETE RESTRICT
  - status_lookup_id → lookup_values(id) ON DELETE SET NULL
索引：
  - shipping_order_number UNIQUE
  - idx_order_id, idx_customer_id, idx_status, idx_shipping_date
欄位：
  - shipping_order_number (varchar 50) -- 出貨單號
  - order_id (bigint)                  -- ★ 關聯到委託確認單
  - customer_id (bigint)
  - shipping_date (date)               -- 出貨日期
  - delivery_method (varchar 100)      -- 配送方式
  - tracking_number (varchar 100)      -- 追蹤號碼
  - consignee_name (varchar 100)       -- 收貨人姓名
  - consignee_address (text)           -- 收貨地址
  - carrier (varchar 100)              -- 承運商
  - status (varchar 50)
  - status_lookup_id (bigint)
  - notes (text)
```

### 2.3 return_orders (退貨單)
```sql
主鍵：id (bigint)
外鍵：
  - original_shipping_order_id → shipping_orders(id) ON DELETE SET NULL
  - customer_id → customers(id) ON DELETE RESTRICT
  - status_lookup_id → lookup_values(id) ON DELETE SET NULL
索引：
  - return_order_number UNIQUE
  - idx_original_shipping_order_id, idx_customer_id, idx_return_date
欄位：
  - return_order_number (varchar 50)        -- 退貨單號
  - original_shipping_order_id (bigint)     -- ★ 原出貨單ID
  - customer_id (bigint)
  - return_date (date)                      -- 退貨日期
  - return_reason (text)                    -- 退貨原因
  - processing_status (varchar 50)          -- 處理狀態
  - status_lookup_id (bigint)
  - notes (text)
```

### 2.4 return_order_items (退貨單明細)
```sql
主鍵：id (bigint)
外鍵：
  - return_order_id → return_orders(id) ON DELETE CASCADE
  - order_item_id → order_items(id) ON DELETE RESTRICT  ★★★ 關鍵！
索引：
  - idx_order_item_id
欄位：
  - return_order_id (bigint)
  - order_item_id (bigint)      -- ★★★ 直接連結到原始訂單品項
  - returned_quantity (decimal)  -- 退貨數量
  - returned_unit (varchar 50)   -- 退貨單位
```

### 2.5 shipping_order_items (出貨單明細)
```sql
主鍵：id (bigint)
外鍵：
  - shipping_order_id → shipping_orders(id) ON DELETE CASCADE
  - order_item_id → order_items(id) ON DELETE RESTRICT
  - inventory_item_id → inventory_items(id)
欄位：
  - shipping_order_id (bigint)
  - order_item_id (bigint)      -- 連結到原始訂單品項
  - inventory_item_id (bigint)  -- 庫存品項
  - shipped_quantity (decimal)   -- 出貨數量
  - shipped_unit (varchar 50)    -- 出貨單位
```

## 三、追溯查詢實作

### 3.1 方案一：透過主單關聯追溯（推薦）

```sql
-- 從退貨單追溯到委託確認單（主單層級）
SELECT
    ro.return_order_number AS '退貨單號',
    ro.return_date AS '退貨日期',
    ro.return_reason AS '退貨原因',
    so.shipping_order_number AS '出貨單號',
    so.shipping_date AS '出貨日期',
    o.order_number AS '委託確認單號',
    o.order_date AS '訂單日期',
    c.customer_name AS '客戶名稱'
FROM return_orders ro
LEFT JOIN shipping_orders so ON ro.original_shipping_order_id = so.id
LEFT JOIN orders o ON so.order_id = o.id
LEFT JOIN customers c ON ro.customer_id = c.id
WHERE ro.id = ? AND ro.deleted_at IS NULL;
```

### 3.2 方案二：透過明細品項追溯（詳細）

```sql
-- 從退貨單明細追溯到原始訂單品項
SELECT
    ro.return_order_number AS '退貨單號',
    roi.returned_quantity AS '退貨數量',
    roi.returned_unit AS '退貨單位',
    oi.product_name AS '產品名稱',
    oi.quantity AS '原訂購數量',
    oi.unit AS '原訂購單位',
    o.order_number AS '委託確認單號',
    o.order_date AS '訂單日期'
FROM return_order_items roi
INNER JOIN return_orders ro ON roi.return_order_id = ro.id
INNER JOIN order_items oi ON roi.order_item_id = oi.id
INNER JOIN orders o ON oi.order_id = o.id
WHERE ro.id = ? AND ro.deleted_at IS NULL;
```

### 3.3 完整追溯（含出貨記錄）

```sql
-- 完整的三層追溯：委託確認單 → 出貨單 → 退貨單
SELECT
    o.order_number AS '委託確認單號',
    o.order_date AS '訂單日期',
    oi.product_name AS '產品名稱',
    oi.quantity AS '訂購數量',

    so.shipping_order_number AS '出貨單號',
    so.shipping_date AS '出貨日期',
    soi.shipped_quantity AS '出貨數量',

    ro.return_order_number AS '退貨單號',
    ro.return_date AS '退貨日期',
    roi.returned_quantity AS '退貨數量',
    roi.return_reason AS '退貨原因',

    c.customer_name AS '客戶名稱'

FROM order_items oi
INNER JOIN orders o ON oi.order_id = o.id
LEFT JOIN shipping_order_items soi ON oi.id = soi.order_item_id
LEFT JOIN shipping_orders so ON soi.shipping_order_id = so.id AND so.deleted_at IS NULL
LEFT JOIN return_order_items roi ON oi.id = roi.order_item_id
LEFT JOIN return_orders ro ON roi.return_order_id = ro.id AND ro.deleted_at IS NULL
LEFT JOIN customers c ON o.customer_id = c.id

WHERE oi.id = ?  -- 以訂單品項為中心
ORDER BY o.order_date, so.shipping_date, ro.return_date;
```

### 4.2 API 端點設計

#### 新增追溯查詢端點

**檔案**：`api/return_orders/get_traceability.php`

```php
<?php
/**
 * 取得退貨單追溯資訊
 * GET /api/return_orders/get_traceability.php?id={return_order_id}
 */
require_once __DIR__ . '/../bootstrap.php';

$return_order_id = $_GET['id'] ?? null;

if (!$return_order_id) {
    echo json_encode(['success' => false, 'message' => '缺少退貨單ID']);
    exit;
}

try {
    $sql = "
        SELECT
            ro.return_order_number,
            ro.return_date,
            ro.return_reason,
            so.id AS shipping_order_id,
            so.shipping_order_number,
            so.shipping_date,
            o.id AS order_id,
            o.order_number,
            o.order_date,
            c.customer_name
        FROM return_orders ro
        LEFT JOIN shipping_orders so ON ro.original_shipping_order_id = so.id
        LEFT JOIN orders o ON so.order_id = o.id
        LEFT JOIN customers c ON ro.customer_id = c.id
        WHERE ro.id = :id AND ro.deleted_at IS NULL
    ";

    $stmt = $pdo->prepare($sql);
    $stmt->execute(['id' => $return_order_id]);
    $result = $stmt->fetch(PDO::FETCH_ASSOC);

    if (!$result) {
        echo json_encode(['success' => false, 'message' => '找不到退貨單']);
        exit;
    }

    echo json_encode(['success' => true, 'data' => $result]);

} catch (Exception $e) {
    error_log("Error in get_traceability.php: " . $e->getMessage());
    echo json_encode(['success' => false, 'message' => '系統錯誤']);
}
```

#### 修改現有 get.php 加入追溯資訊

**檔案**：`api/return_orders/get.php`

在現有查詢中加入 JOIN：

```php
// 原本的查詢
```

```javascript
function showReturnOrderDetails(id) {
    fetch(`api/return_orders/get.php?id=${id}`)
        .then(response => response.json())
        .then(result => {
            if (result.success) {
                const data = result.data;

                // 顯示追溯資訊
                displayTraceability(data);

                // 原有的詳情顯示邏輯...
            }
        });

function displayTraceability(data) {
    // 委託確認單
    const orderNumber = document.getElementById('original-order-number');
    const orderDate = document.getElementById('original-order-date');
    if (data.order_number) {
        orderNumber.textContent = data.order_number;
        orderDate.textContent = data.order_date || '';
        // 可點擊前往查看
        orderNumber.closest('a').dataset.orderId = data.order_id;
    } else {
        orderNumber.textContent = '—';
        orderDate.textContent = '無原始訂單';
    }

    // 出貨單
    const shippingNumber = document.getElementById('shipping-order-number');
    const shippingDate = document.getElementById('shipping-date');
    if (data.shipping_order_number) {
        shippingNumber.textContent = data.shipping_order_number;
        shippingDate.textContent = data.shipping_date || '';
        shippingNumber.closest('a').dataset.shippingOrderId = data.shipping_order_id;
    } else {
        shippingNumber.textContent = '—';
        shippingDate.textContent = '無出貨記錄';
    }

    // 退貨單（當前）
    document.getElementById('return-order-number').textContent = data.return_order_number;
    document.getElementById('return-date').textContent = data.return_date;
}

// 追溯連結點擊事件
document.addEventListener('click', (e) => {
    if (e.target.closest('[data-action="view-order"]')) {
        e.preventDefault();
        const orderId = e.target.closest('a').dataset.orderId;
        if (orderId) {
            // 切換到委託確認單頁面並開啟該訂單

            // 等待頁面載入後開啟詳情

                if (typeof window.viewOrderDetails === 'function') {
                    window.viewOrderDetails(orderId);
                }
            }, 500);
        }
    }

    if (e.target.closest('[data-action="view-shipping-order"]')) {
        e.preventDefault();
        const shippingOrderId = e.target.closest('a').dataset.shippingOrderId;
        if (shippingOrderId) {
            window.loadTab('shipping_orders', '出貨單管理');
            setTimeout(() => {
                if (typeof window.viewShippingOrderDetails === 'function') {
                    window.viewShippingOrderDetails(shippingOrderId);
                }
            }, 500);
        }
    }
});
```

## 五、建議改進方向

### 5.1 資料完整性檢查

目前 `shipping_orders.order_id` 和 `return_orders.original_shipping_order_id` 都是 `DEFAULT NULL`，這可能導致：

1. **出貨單無法追溯到委託確認單**
2. **退貨單無法追溯到出貨單**

**建議**：
- 將 `shipping_orders.order_id` 改為 `NOT NULL`（出貨單必須有來源訂單）
- 考慮將 `return_orders.original_shipping_order_id` 改為 `NOT NULL`（退貨單必須有來源出貨單）

```sql
-- 資料完整性改進（需先確保現有資料已填入）
ALTER TABLE shipping_orders
MODIFY order_id bigint NOT NULL COMMENT '訂單ID（必填）';

ALTER TABLE return_orders
MODIFY original_shipping_order_id bigint NOT NULL COMMENT '原出貨單ID（必填）';
```

### 5.2 新增欄位建議

在 `return_orders` 表中可考慮新增：

```sql
ALTER TABLE return_orders
ADD COLUMN is_complete_return TINYINT(1) DEFAULT 0 COMMENT '是否全部退貨',
ADD COLUMN received_date DATE COMMENT '收到退貨日期',
ADD COLUMN inspection_status VARCHAR(50) COMMENT '檢驗狀態',
ADD COLUMN refund_amount DECIMAL(14,2) COMMENT '退款金額';
```

### 5.3 建立視圖（View）簡化查詢

```sql
CREATE OR REPLACE VIEW v_return_orders_traceability AS
SELECT
    ro.id AS return_order_id,
    ro.return_order_number,
    ro.return_date,
    ro.return_reason,
    ro.processing_status,

    so.id AS shipping_order_id,
    so.shipping_order_number,
    so.shipping_date,

    o.id AS order_id,
    o.order_number,
    o.order_date,

    c.id AS customer_id,
    c.customer_name,
    c.contact_person

FROM return_orders ro
LEFT JOIN shipping_orders so ON ro.original_shipping_order_id = so.id
LEFT JOIN orders o ON so.order_id = o.id
LEFT JOIN customers c ON ro.customer_id = c.id
WHERE ro.deleted_at IS NULL;
```

使用視圖後，查詢變得非常簡單：

```sql
SELECT * FROM v_return_orders_traceability WHERE return_order_id = ?;
```

## 七、總結

### 現有設計優點

✅ **完整的外鍵約束**：確保資料完整性
✅ **明細層級直接連結**：`return_order_items.order_item_id` 直接連到 `order_items.id`
✅ **軟刪除支援**：使用 `deleted_at` 保留歷史記錄
✅ **索引完整**：所有關聯欄位都有索引

### 可改進之處

⚠️ **關聯欄位允許 NULL**：可能導致追溯斷鏈
⚠️ **缺少業務欄位**：如退款金額、檢驗狀態等
⚠️ **缺少統計視圖**：需手動 JOIN 三層查詢

### 實作優先順序

1. **高優先**：實作 API 端點 `get_traceability.php`
2. **高優先**：在退貨單詳情頁顯示追溯資訊
3. **中優先**：實作追溯連結跳轉功能
4. **中優先**：建立視圖簡化查詢
5. **低優先**：資料完整性改進（需評估現有資料）
6. **低優先**：新增業務欄位（需業務確認）
