# 庫存出貨履歷追蹤改善方案

> 日期：2026-02-08
> 狀態：**待實作**

---

## 一、問題背景

使用者反映庫存異動紀錄（`inventory_transactions`）、庫存品項（`inventory_items`）、出貨紀錄（`shipping_orders` / `shipping_order_items`）三者之間缺乏連動，無法完整追蹤從入庫到出貨的履歷。

---

### 2.2 關聯圖

```

     │ 完工入庫
     ▼
inventory_items              inventory_transactions

```

## 三、發現的嚴重缺陷

### 🔴 嚴重（連動斷裂）

| 代號 | 問題 | 位置 | 影響 |
|------|------|------|------|
| **G1** | 出貨確認（→shipped）時**不扣減** `quantity_on_hand` | `api/shipping_orders/update.php` | 庫存數量永遠不因出貨減少 |
| **G2** | 出貨確認時**不增加** `quantity_shipped` | `api/shipping_orders/update.php` | 詳情頁 quantity_shipped 永遠是 0 |
| **G3** | 出貨確認時**不更新** `inventory_items.status` | `api/shipping_orders/update.php` | 全數出完仍顯示「在庫」 |
| **G4** | 出貨時**不建立** `inventory_transactions` 記錄 | `api/shipping_orders/update.php` | 異動紀錄頁面完全看不到出貨歷程 |

### 🟡 中度

| 代號 | 問題 | 位置 | 影響 |
|------|------|------|------|
| **G5** | `direction` 值不一致：`inventory_items/update.php` 用 `in`/`out`，lookup 定義 `inbound`/`outbound` | `api/inventory_items/update.php` | 異動記錄方向顯示可能異常 |
| **G6** | `add_item.php` 在 `draft` 階段就更新 `order_items.total_shipped_quantity` | `api/shipping_orders/add_item.php` | 草稿階段提前計入訂單出貨統計 |
| **G7** | `delete.php` 釋放 allocated 時查詢用 `quantity` 欄位名（不存在），應為 `shipped_quantity` | `api/shipping_orders/delete.php` | 刪除出貨單無法正確釋放配貨數 |
| **G8** | 後端 `helpers.php` 的 `$allowedStatuses` 與前端不一致，後端只有 `['draft', 'pending', 'shipped', 'delivered', 'cancelled']`，缺少 `confirmed`/`preparing`/`packed` | `api/shipping_orders/helpers.php` | 狀態驗證不完備 |
| **G9** | `quantity_allocated` 增減邏輯不對稱：加入時增、出貨確認時未轉換為 shipped | `add_item.php` vs `update.php` | allocated 數字永遠累加不清除 |

## 四、改善方案

### 4.1 完整生命週期流程

```
┌──────────────────────────────────────────────────────────────┐
│                      出貨生命週期                             │
│                                                              │
│  ① 加入出貨單 (draft)                                        │
│     inventory_items.quantity_allocated += qty                 │
│     inventory_items.status → 'allocated' (若有配貨)           │
│     ⚠ order_items.total_shipped_quantity 暫不更新              │
│                                                              │
│  ② 出貨確認 (→ shipped)                                      │
│     inventory_items.quantity_on_hand -= qty                   │
│     inventory_items.quantity_allocated -= qty                 │
│     inventory_items.quantity_shipped += qty                   │
│     inventory_items.status → shipped/allocated/in_stock       │
│     inventory_transactions: ref=shipping_order, dir=outbound  │
│     order_items.total_shipped_quantity += qty  ← 此時才確認    │
│                                                              │
│  ③ 退貨 (return)                                             │
│     inventory_items.quantity_on_hand += return_qty            │
│     inventory_items.quantity_shipped -= return_qty            │
│     inventory_items.status → in_stock (如有庫存回來)           │
│     inventory_transactions: ref=return_order, dir=inbound     │
│                                                              │
│  ④ 取消出貨 (→ cancelled / 刪除)                              │
│     inventory_items.quantity_allocated -= qty                 │
│     inventory_items.status → 重新計算                         │
│     inventory_transactions: 反向沖銷                          │
└──────────────────────────────────────────────────────────────┘
```

### 4.3 核心函式：recalculateInventoryStatus

```php
/**
 * 重新計算庫存品項狀態
 * 根據 quantity_on_hand、quantity_allocated、quantity_shipped 自動判斷狀態
 */
function recalculateInventoryStatus($pdo, $inventoryItemId) {
    $stmt = $pdo->prepare("
        SELECT quantity_on_hand, quantity_allocated, quantity_shipped
        FROM inventory_items
        WHERE id = ? AND deleted_at IS NULL
    ");
    $stmt->execute([$inventoryItemId]);
    $item = $stmt->fetch(PDO::FETCH_ASSOC);

    if (!$item) return null;

    if ($item['quantity_on_hand'] <= 0 && $item['quantity_shipped'] > 0) {
        $newStatus = 'shipped';      // 全部出貨完畢
    } elseif ($item['quantity_allocated'] > 0) {
        $newStatus = 'allocated';    // 有配貨中的數量
    } else {
        $newStatus = 'in_stock';     // 正常在庫
    }

    $pdo->prepare("UPDATE inventory_items SET status = ? WHERE id = ?")
        ->execute([$newStatus, $inventoryItemId]);

    return $newStatus;
}
```

### 4.4 出貨確認核心邏輯（update.php 新增）

```php
// 當 status 轉換為 'shipped' 時，執行庫存連動
if ($oldStatus !== 'shipped' && $newStatus === 'shipped') {

    // 取得所有出貨品項
    $itemsStmt = $pdo->prepare("
        SELECT soi.*, ii.quantity_on_hand, ii.inventory_number
        FROM shipping_order_items soi
        JOIN inventory_items ii ON ii.id = soi.inventory_item_id
        WHERE soi.shipping_order_id = ?
    ");
    $itemsStmt->execute([$id]);
    $shippingItems = $itemsStmt->fetchAll(PDO::FETCH_ASSOC);

    foreach ($shippingItems as $si) {
        $qty = $si['shipped_quantity'];
        $invId = $si['inventory_item_id'];
        $newOnHand = $si['quantity_on_hand'] - $qty;

        // 1. 更新庫存數量
        $pdo->prepare("
            UPDATE inventory_items SET
                quantity_on_hand = quantity_on_hand - ?,
                quantity_allocated = GREATEST(0, quantity_allocated - ?),
                quantity_shipped = quantity_shipped + ?

                 notes, created_by_employee_id)
            VALUES (?, ?, ?,
                (SELECT work_order_id FROM inventory_items WHERE id = ?),
                'shipping_order', ?, 'outbound', ?, ?,
                CONCAT('出貨單 ', ?, ' 出貨'),
                ?)

            $invId,
            $shippingOrder['order_id'],
            $si['order_item_id'],

```

### 4.5 庫存異動記錄格式

出貨確認時建立的 `inventory_transactions`：

| 欄位 | 值 |
|------|-----|
| `inventory_item_id` | 對應庫存品項 ID |
| `order_id` | 出貨單的訂單 ID |
| `order_item_id` | 出貨品項對應的訂單品項 ID |
| `work_order_id` | 從庫存品項帶入 |
| `ref_type` | `shipping_order` |
| `ref_id` | 出貨單 ID |
| `direction` | `outbound` |
| `quantity` | 出貨數量 |

## 六、驗證要點

### 6.1 功能測試

1. **入庫 → 加入出貨單**
   - [ ] `quantity_allocated` 正確增加
   - [ ] `status` 變為 `allocated`
   - [ ] `order_items.total_shipped_quantity` 不應在此時更新

2. **出貨確認 (→ shipped)**
   - [ ] `quantity_on_hand` 正確減少
   - [ ] `quantity_allocated` 正確減少
   - [ ] `quantity_shipped` 正確增加
   - [ ] `status` 正確更新（全出 → `shipped`，部分出 → `allocated`/`in_stock`）
   - [ ] `inventory_transactions` 有一筆 `outbound` + `shipping_order` 記錄
   - [ ] `order_items.total_shipped_quantity` 此時才更新

3. **刪除出貨品項 / 取消出貨**
   - [ ] `quantity_allocated` 正確釋放
   - [ ] `status` 正確回算

4. **庫存異動頁面**
   - [ ] 可看到出貨相關的 `outbound` 記錄
   - [ ] 方向顯示正確（入庫/出庫/調整）

### 6.2 數據一致性公式

```
quantity_on_hand + quantity_shipped + quantity_allocated(已確認出貨的)
    = total_good_units（不考慮手動調整的情況下）

或更精確：
quantity_on_hand = total_good_units - quantity_shipped - 手動出庫調整量
```
