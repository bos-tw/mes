## P0：資料正確性

- [ ] 建立部分入庫彙總共用 helper，統一一般／拆分工單口徑。
- [ ] `show.php` 回傳 `partial_receipt_summary`。
- [ ] `show.php` 回傳 `partial_receipts[]` 歷程明細。
- [ ] 彙總部分入庫關聯庫存的已出貨及尚在庫數量。
- [ ] 確認重量與支數取整規則。
- [ ] 一般工單結案扣除有效部分入庫。
- [ ] 拆分工單結案扣除有效部分入庫。
- [ ] 部分入庫超過實際良品時阻擋結案。
- [ ] 最終剩餘量為 0 時不建立零數量庫存。
- [ ] 結案後將有效部分入庫標為 `settled`。
- [ ] 工單、庫存、出貨的合計可互相回推。

## 必測案例

- [ ] 一般工單 100 kg，部分入庫 30 + 20 kg，結案只補 50 kg。
- [ ] 部分入庫 30 kg、已出貨 10 kg，畫面顯示在庫 20 kg。
- [ ] 已出貨 10 kg 不列入工單短缺。
- [ ] 拆分機台完成 40 kg、已入庫 30 kg，只能再入庫 10 kg。
- [ ] 拆分機台合計不可超過主工單。
- [ ] 待開始工單不可部分入庫。
- [ ] 已完成工單不可部分入庫。
- [ ] 已有正式庫存不可部分入庫。
- [ ] 缺少本次淨重回傳 400。
- [ ] 超過剩餘淨重回傳 409。
- [ ] 未出貨部分入庫可合法沖銷。
- [ ] 已配貨部分入庫不可直接沖銷。
- [ ] 已出貨部分入庫不可直接沖銷。
- [ ] 結案時部分入庫超過良品則完整回滾。
- [ ] 桌面、手機、報表彙總一致。
