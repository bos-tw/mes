# 專案架構混亂問題分析報告

> 生成日期：2026-02-04
> 觸發原因：修改 customers.html 無效，發現架構仍有混亂

---

## 🔍 問題根源

### 您遇到的具體問題

**症狀**：修改 `modules/customers.html` 中的 Modal 標題無效

**根本原因**：
1. ✅ `customers` 模組**已完成配置化遷移**（2026-01-30）
2. ✅ 配置檔位於 `core/configs/customers.config.js`
3. ✅ Modal 標題由配置檔定義，HTML 中的標題會被配置覆蓋
4. ❌ 但 HTML 檔案**仍然存在且未標註已配置化**

## ⚠️ 核心問題：雙重架構並存

### 問題 1：HTML 檔案殘留但未使用

**已配置化的模組仍保留 HTML 檔案**

```
modules/
├── customers.html         ❌ 已配置化，此檔案不再使用
├── companies.html         ❌ 已配置化，此檔案不再使用
├── departments.html       ❌ 已配置化，此檔案不再使用
├── employees.html         ❌ 已配置化，此檔案不再使用
...（共 41 個已配置化但 HTML 仍存在）
```

**造成的混亂**：
- ❌ 開發者不知道該改哪個檔案
- ❌ 修改 HTML 無效，因為實際使用配置檔
- ❌ 無法從檔案系統判斷模組是否已配置化

### 問題 2：配置檔位置不顯眼

**配置檔在 `core/configs/` 目錄**

```
core/configs/
├── customers.config.js    ← 實際控制 customers 模組
├── companies.config.js    ← 實際控制 companies 模組
...
```

**造成的混亂**：
- ❌ 開發者習慣直接找 `modules/*.html`
- ❌ 不知道配置檔的存在
- ❌ 不清楚哪些模組已配置化

### 問題 3：混合模式增加複雜度

**部分模組使用混合配置模式**

| 模組 | 模式 | 說明 |
|------|------|------|
| `work_orders` | 混合 | 頁面結構配置化，Modal 從 HTML 載入 |
| `order_items` | 混合 | 頁面結構配置化，Modal 從 HTML 載入 |
| `production_quality_records` | 混合 | 部分配置化 |
| `inventory_items` | 混合 | 含 customModalHtml |
| `audit_logs` | 混合 | dataPrefix 特殊處理 |

**造成的混亂**：
- ❌ 需要同時維護配置檔和 HTML
- ❌ 難以判斷哪部分由配置控制
- ❌ 新開發者學習曲線陡峭

## 🎯 解決方案建議

### 方案 A：檔案標記系統（推薦）

**目標**：讓開發者立即知道模組狀態

#### 1. 重新命名已配置化的 HTML

```bash
# 將已配置化的 HTML 改名為 .bak
modules/customers.html → modules/customers.html.bak
modules/companies.html → modules/companies.html.bak
modules/departments.html → modules/departments.html.bak
...
```

#### 2. 在已配置化的 HTML 頂部加入警告

```html
<!-- ⚠️ 此檔案已不再使用 - 模組已配置化 -->
<!-- 配置檔位置: core/configs/customers.config.js -->
<!-- 如需修改，請編輯配置檔，不要修改此 HTML -->

<div data-module="customers">
...
```

#### 3. 建立索引檔案

建立 `modules/README.md`：

```markdown
# 模組檔案索引

## ✅ 已配置化模組（不使用 HTML 檔案）

| 模組 | 配置檔 | HTML 狀態 |
|------|--------|----------|
| customers | core/configs/customers.config.js | ⚠️ .html.bak 僅供參考 |
| companies | core/configs/companies.config.js | ⚠️ .html.bak 僅供參考 |
...

## 📝 修改指引

- **已配置化模組**：編輯 `core/configs/*.config.js`
- **未配置化模組**：編輯 `modules/*.html`
- **混合模式**：查看配置檔中的 `requiresHtmlModal` 說明
```

### 方案 B：目錄重組（大規模）

**目標**：從檔案結構清楚區分

```
modules/
├── configurable/          # 已配置化（只保留參考）
│   ├── customers.html.ref
│   ├── companies.html.ref
│   └── README.md         # 說明這些檔案僅供參考
├── hybrid/               # 混合模式
│   ├── work_orders.html  # Modal 部分仍使用
│   ├── order_items.html
│   └── README.md         # 說明哪部分由配置控制
└── legacy/               # 未配置化（如果有）
    └── README.md
```
