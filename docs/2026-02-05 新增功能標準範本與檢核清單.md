# 新增功能標準範本與檢核清單

> 📅 建立日期：2026-02-05
> 🎯 目的：讓後續新增模組遵循統一規範，避免到處修改與風格分裂

---

## 1) 新模組建立標準流程

1. **確認是否配置化**
   - 有配置檔 → 改 config
   - 無配置檔 → 建立 config + 使用 ModuleRenderer

2. **若需混合模式**
   - 在 config 設定 requiresHtmlModal: true
   - 建立 modules/{module}.html（只放 Modal）

3. **建立前後驗證**
   - 修改前：node tools/validate-config-modules.js
   - 修改後：node tools/validate-config-modules.js

---

## 2) 配置檔範本（標準 CRUD）
```javascript
ModuleConfig.register('your_module', {
    title: '模組標題',
    subtitle: '模組描述',

    actions: [
        { label: '新增', icon: 'fa-plus', action: 'create', style: 'primary' }
    ],

    filters: [
        { name: 'keyword', label: '關鍵字', type: 'text', placeholder: '搜尋...' }
    ],

    columns: [
        { key: 'id', label: 'ID', sortable: true, selectable: true },
        { key: 'name', label: '名稱', sortable: true, selectable: true },
        { key: 'actions', label: '操作', sortable: false, selectable: false }
    ],

    modal: {
        title: '新增資料',
        size: 'medium',
        hiddenFields: ['id'],
        sections: [
            {
                title: '基本資訊',
                fields: [
                    { name: 'name', label: '名稱', type: 'text', required: true }
                ]
            }
        ]
    }
});
```

## 3) JS 標準範本
```javascript
function setFieldValue(name, value) {
    const field = form.querySelector(`[name="${name}"]`);
    if (field) {
        if (field.type === 'checkbox') {
            field.checked = !!value;
        } else {
            field.value = value || '';
        }
    } else {
        console.warn(`your_module: 欄位不存在 - ${name}`);
    }
}
```

## 5) 檢核清單（新增功能必過）

### ✅ 配置檔
- [ ] modal 有 `hiddenFields: ['id']`
- [ ] filters / columns / actions 格式一致
- [ ] 若 requiresHtmlModal，HTML 檔案存在

### ✅ JS
- [ ] 使用 setFieldValue 防禦性賦值
- [ ] 無直接 querySelector().value =
- [ ] 事件委派集中於 moduleRoot

### ✅ UI
- [ ] 按鈕類別符合規範
- [ ] Modal 尺寸使用標準值
- [ ] inline-label / full-width 正確

### ✅ 驗證
- [ ] 修改前後執行 validate-config-modules.js
- [ ] 驗證結果 0 錯誤
