   - 點擊「存草稿」再次儲存
   - ✅ 應該正常更新

3. **測試草稿發布**：
   - 編輯某個草稿
   - 補齊「發佈對象」（必填）
   - **點擊「發佈」按鈕**
   - ✅ 應該顯示「公告發佈成功」
   - ✅ 自動切換到「我發布的」分頁

4. **測試草稿不影響收件匣**：

   - ✅ 未讀數量不應計算草稿

## 🔍 預期功能

### 通知中心

| 分頁 | 顯示內容 | status 篩選 | 權限 |
|------|----------|-------------|------|
| 收到的通知 | 目標包含自己的已發布通知 | published | 一般使用者 |
| 我發布的 | 自己發布的已發布通知 | published + created_by_me | 管理員 |
| **草稿匣** | **自己建立的草稿** | **draft + created_by_me** | **管理員** |
| 歷史紀錄 | 包含已過期的已發布通知 | published + include_expired | 一般使用者 |

### 留言系統

| 資料夾 | 顯示內容 | status 篩選 | 備註 |
|--------|----------|-------------|------|
| 收件匣 | 自己收到的留言 | sent（只顯示已發送） | 不含草稿 |
| 寄件匣 | 自己發送的留言 | sent | 不含草稿 |
| **草稿匣** | **自己建立的草稿** | **draft** | **可無收件人** |
| 垃圾桶 | 已刪除的留言 | - | 軟刪除 |

## ✨ 新功能特點

1. **草稿自動儲存位置**：
   - 通知：新增後自動切換到「草稿匣」分頁
   - 留言：新增後自動切換到「草稿匣」資料夾

2. **草稿寬鬆驗證**：
   - 通知草稿可以不選擇「發佈對象」
   - 留言草稿可以不選擇「收件人」

3. **草稿隔離性**：
   - 草稿只有建立者可見
   - 草稿不會出現在他人的收件匣
   - 未讀數量不計算草稿

4. **草稿發布流程**：
   - 編輯草稿 → 補齊必填欄位 → 點「發佈/發送」
   - 發布後 status 自動改為 `published/sent`
   - 留言發布時才建立 `message_recipients` 記錄

---

## 🚨 已知限制

1. **草稿無定時自動儲存**：  
   - 目前需要手動點「存草稿」按鈕
   - 未來可考慮加入 Auto-save 功能

2. **草稿編輯為發布無 API**：  
   - 目前草稿編輯使用 `update.php`，狀態變更需要帶 `status` 欄位
   - 建議未來加入專用的「發布草稿」API endpoint

3. **留言草稿無收件人清單**：  
   - 草稿儲存時若選擇了收件人，編輯時無法顯示（因為沒有建立 `message_recipients`）
   - 建議未來加入 `draft_recipients` JSON 欄位暫存
