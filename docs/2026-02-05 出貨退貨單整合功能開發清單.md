## 🐛 Bug 修復記錄（2026-02-05）

### 發現的問題

| # | 問題 | 錯誤訊息 | 狀態 |
|---|------|----------|------|
| 1 | switchToTab 函數不存在 | `window.switchToTab is not a function` | ✅ 已修復 |

| 3 | disabled select 值未收集 | 建立退貨單失敗無錯誤訊息 | ✅ 已修復 |

### 修復內容

#### Bug #1：switchToTab 函數不存在
- **位置**：`js/shipping_orders.js:841`、`js/return_orders.js:814`
- **原因**：程式碼使用了不存在的 `window.switchToTab`，系統實際使用 `window.openTab`
- **修復**：改用 `window.openTab('module_id', 'title', 'modules/xxx.html')`

#### Bug #2：SQL 參數綁定錯誤
- **位置**：`api/return_orders/create_from_shipping.php:305-325`
- **原因**：原本使用 `GROUP BY soi.id` 導致每行只返回單一項目統計
- **修復**：使用子查詢先計算每個品項的退貨總量，再進行彙總

#### Bug #3：disabled select 值未收集
- **位置**：`js/return_orders.js:430-433`
- **原因**：disabled 的 `<select>` 欄位不會被 `FormData` 收集
- **修復**：在 `saveData()` 中手動收集 disabled select 的值

#### Bug #4：表單 required 衝突
- **位置**：`modules/return_orders.html:31`
- **原因**：`original_shipping_order_id` 同時設定 `required` 和 `disabled`
- **修復**：移除 `required` 屬性（該欄位是選填的）
