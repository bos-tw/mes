# 更新安裝包打包流程

> 適用專案：MES（`C:\Apache24\htdocs\mes`）
> 
> 目的：將指定程式檔與 migration SQL 打成可上傳的更新 ZIP。

## 1. 前置條件

1. 以 PowerShell 開啟終端。
2. 進入專案根目錄：

```powershell
Set-Location "C:\Apache24\htdocs\mes"
```

3. 若本次視窗尚未允許執行腳本，先執行：

```powershell
Set-ExecutionPolicy -Scope Process Bypass -Force
```

## 2. 建立版本更新說明檔（release note）

```powershell
if (!(Test-Path "release-notes")) { New-Item -ItemType Directory -Path "release-notes" | Out-Null }

@(
"1. 新增系統更新初始化檢查按鈕。"
"2. 新增更新包上傳/套用/狀態 API。"
"3. 補強缺表與錯誤訊息處理。"
) | Set-Content -Path "release-notes\2026-05-09-v1.0.2.txt" -Encoding UTF8
```

## 3. 打包（建議用 splatting，避免參數拆行失敗）

```powershell
$params = @{
    VersionNumber     = "v1.0.2"
    FileVersion       = "v1.0.2"
    ReleaseDate       = "2026-05-09"
    ChangeSummaryFile = "release-notes/2026-05-09-v1.0.2.txt"
    Files             = @(
        "modules/security_settings.html",
        "js/security_settings.js",
        "styles.css",
        "api/system_update_common.php",
        "api/system_update_upload.php",
        "api/system_update_apply.php",
        "api/system_update_status.php",
        "api/system_update_init_check.php",
        "api/system_update_history.php",
        "index.php",
        "script.js"
    )
    Migrations        = @(
        "migrations/2026_05_09_create_system_update_jobs.sql",
        "migrations/2026_05_09_create_system_update_logs.sql"
    )
    OutputDir         = "dist"
}

& ".\tools\build-update-package.ps1" @params
```

## 4. 驗證輸出檔

```powershell
Get-ChildItem ".\dist\update_v1.0.2_*.zip" |
    Sort-Object LastWriteTime -Descending |
    Select-Object -First 1 FullName, LastWriteTime, Length
```

成功時會在 `dist/` 看到類似：
- `update_v1.0.2_20260509_182554.zip`

## 5. 可選：單行版指令

> 想一次貼上一行可用此版本：

```powershell
Set-Location "C:\Apache24\htdocs\mes"; if (!(Test-Path "release-notes")) { New-Item -ItemType Directory -Path "release-notes" | Out-Null }; @("1. 新增系統更新初始化檢查按鈕。","2. 新增更新包上傳/套用/狀態 API。","3. 補強缺表與錯誤訊息處理。") | Set-Content -Path "release-notes\2026-05-09-v1.0.2.txt" -Encoding UTF8; & ".\tools\build-update-package.ps1" -VersionNumber "v1.0.2" -FileVersion "v1.0.2" -ReleaseDate "2026-05-09" -ChangeSummaryFile "release-notes/2026-05-09-v1.0.2.txt" -Files @("modules/security_settings.html","js/security_settings.js","styles.css","api/system_update_common.php","api/system_update_upload.php","api/system_update_apply.php","api/system_update_status.php","api/system_update_init_check.php","api/system_update_history.php","index.php","script.js") -Migrations @("migrations/2026_05_09_create_system_update_jobs.sql","migrations/2026_05_09_create_system_update_logs.sql") -OutputDir "dist"
```

## 6. 常見錯誤與排除

### 錯誤 1：`-FileVersion is not recognized`

原因：上一行腳本已結束，下一行直接輸入 `-FileVersion ...` 會被當成獨立命令。

解法：
1. 把整段命令一次貼上執行。
2. 或使用本文件第 3 節的 `splatting` 寫法。

### 錯誤 2：`build-update-package.ps1 is not recognized`

原因：沒有使用腳本路徑呼叫。

解法：
```powershell
& ".\tools\build-update-package.ps1" ...
```
