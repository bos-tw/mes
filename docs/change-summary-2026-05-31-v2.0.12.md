# v2.0.12 更新摘要（2026-05-31）

本更新包重新彙整本輪對話期間的全部工作區異動，並補入最新 `DEVELOPMENT_PROGRESS_SUMMARY.md` 交接摘要，供遠端一鍵更新與下一輪開發接手。

## 核心更新
- 生產工單支援一般/拆分工單流程調整與介面優化。
- 拆分機台流程、機台頁籤、排程節點與機台明細資料寫入/查詢強化。
- 部分入庫流程改為工單層共用入口（一般工單/拆分工單可依規則使用）。
- 拆分機台區保留統計與來源資訊，避免雙入口操作混淆。
- 拆分工單不良紀錄欄位補齊為完整表格欄位（含公差/PPM/備註/不良數量）。
- 一般工單與拆分工單的篩分服務明細皆移除「服務名稱(客製)」欄位以避免重複。

## API/資料一致性與防呆
- `api/work_orders/*` 多處修正，包含儲存、查詢、部分入庫、排程節點、機台明細相容處理。
- 欄位存在檢查與 SQL 相容防護，降低 schema 差異導致的 500 風險。
- 稽核與資料同步檢查腳本更新（`audit-system-health.js`、`audit-data-sync.js`）。

## 資料庫變更
- 包含 migration：`migrations/2026_05_31_add_split_work_order_foundation.sql`
- 修正 `work_order_partial_receipts.machine_run_id` 為可空值，讓一般工單也能使用工單層部分入庫。
- 修正 migration no-op 語句，避免一鍵更新器以 `PDO::exec()` 執行時留下未讀取結果集造成套用失敗。
- 包含 schema 同步腳本更新：`tools/sync-local-schema.ps1`

- 更新拆分工單實作追蹤文件與討論備忘錄。
